# v0.4.0 Milestone Audit

## Result

**Status:** gaps_found

This milestone is not ready to archive. The main problem is not one isolated code bug. The planning set for v0.4.0 still contains unfinished phases, missing verification files, human-only verification that was never closed, and requirement tracking that drifted out of sync with what the phase reports say.

## Scope Note

This audit used the current `ROADMAP.md`, `REQUIREMENTS.md`, phase `SUMMARY.md`, phase `VERIFICATION.md`, and phase `VALIDATION.md` files under `.planning/phases/`.

The scope file itself is inconsistent:

| Issue | Evidence | Impact |
| --- | --- | --- |
| Merge-conflict residue remains in roadmap | `ROADMAP.md` still contains `<<<<<<< Updated upstream` | Milestone boundaries are not fully trustworthy |
| Duplicate phase numbering exists | `ROADMAP.md` contains two different "Phase 71" entries | Phase counts and requirement counts are inflated/ambiguous |
| Later requirements are not centrally registered | AUTH-01..06, REQ-86-01..04, REQ-87-01..15 are absent from `REQUIREMENTS.md` | Passed work cannot be fully traced in the milestone registry |

Because of that, all scores below are conservative.

## Main Blockers

1. The milestone itself is unfinished. There are 21 phase entries inside the current v0.4.0 scope with no verification at all.
2. The bootstrap/runtime cutover is not closed. Phase 40 has no final verification and BOOT-04 is still pending.
3. The pairing path is not audit-clean. Phases 46, 46.2, and 46.4 still depend on human verification, and Phase 86 still has open gaps.
4. The frontend daemon cutover is not complete. Phases 77 through 82 are still open or unverified.
5. The planning documents no longer form a reliable single source of truth. The roadmap has merge residue and duplicated numbering, and the requirements registry is missing several later requirement sets.

## Recommendation

Do not archive or complete v0.4.0 yet.

The next cleanup order should be:

1. Repair planning truth first: clean `ROADMAP.md`, remove duplicate phase numbering, and register missing requirement IDs in `REQUIREMENTS.md`.
2. Reconcile stale requirement states for phases 72, 73, 84, 85, 87.
3. Close missing verification for foundational runtime phases, especially 36, 40, 45, 46.1, 46.3, 46.5, 46.6.
4. Finish or explicitly defer the open frontend daemon cutover phases 76-82 instead of leaving them inside the active milestone scope.
5. Re-run phase verification for 85 and 86 after their documentation/test issues are actually closed.
