**1. Replace `restore_clipboard_entry` command** (per D-04, D-05):

Replace the current `restore_clipboard_entry` function (lines ~532-538) and `restore_clipboard_entry_impl` helper (lines ~540-620) with a thin daemon proxy. The new command:

- Adds `daemon_connection: State<'_, DaemonConnectionState>` parameter (following setup.rs pattern)
- Keeps `runtime: State<'_, Arc<AppRuntime>>` only for the fallback `forward_clipboard_event` call (the sole frontend update mechanism per research open question 2)
- Calls `DaemonClipboardClient::new(daemon_connection.inner().clone()).restore_clipboard_entry(&entry_id).await`
- On success, emits `forward_clipboard_event` with `ClipboardEvent::NewContent { entry_id, preview: "Clipboard restored", origin: "local" }` — this is the ONLY frontend update mechanism (LocalRestore skips capture in daemon, so no WS `clipboard.new_content` event is emitted)
- Maps errors: parse daemon HTTP status to preserve 404 as `CommandError::NotFound`, other errors as `CommandError::internal` (F-4)

New code:

```rust
/// Restore clipboard entry to system clipboard.
/// Delegates to daemon HTTP API (per D-04) — daemon owns origin tracking and outbound sync.
#[tauri::command]
pub async fn restore_clipboard_entry(
    runtime: State<'_, Arc<AppRuntime>>,
    daemon_connection: State<'_, DaemonConnectionState>,
    entry_id: String,
    _trace: Option<TraceMetadata>,
) -> Result<bool, CommandError> {
    let span = info_span!(
        "command.clipboard.restore_entry",
        trace_id = tracing::field::Empty,
        trace_ts = tracing::field::Empty,
        entry_id = %entry_id,
    );
    record_trace_fields(&span, &_trace);

    async move {
        uc_daemon_client::DaemonClipboardClient::new(daemon_connection.inner().clone())
            .restore_clipboard_entry(&entry_id)
            .await
            .map_err(map_daemon_restore_error)?;

        // Emit frontend event after daemon success.
        // This is the ONLY frontend update — LocalRestore skips capture in daemon,
        // so no WS clipboard.new_content event is emitted (F-2 correction).
        if let Some(app) = runtime.app_handle().as_ref() {
            if let Err(err) = crate::events::forward_clipboard_event(
                app,
                crate::events::ClipboardEvent::NewContent {
                    entry_id: entry_id.clone(),
                    preview: "Clipboard restored".to_string(),
                    origin: "local".to_string(),
                },
            ) {
                tracing::warn!(error = %err, entry_id = %entry_id, "Failed to emit restore event");
            }
        }

        Ok(true)
    }
    .instrument(span)
    .await
}
```

/// Maps daemon restore errors to CommandError, preserving 404 distinction.
/// Used by both restore_clipboard_entry command and tests.
fn map_daemon_restore_error(e: anyhow::Error) -> CommandError {
    let msg = e.to_string();
    if msg.starts_with("[NOT_FOUND]") {
        CommandError::NotFound("Clipboard entry not found".to_string())
    } else {
        CommandError::internal(e)
    }
}
