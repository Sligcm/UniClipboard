Follow the clipboard.rs module pattern exactly.

```rust
pub fn router() -> Router<DaemonApiState> {
    Router::new()
        .route(http_route::SETTINGS, get(get_settings_handler))
        .route(http_route::SETTINGS, put(update_settings_handler))
}
```

- `get_settings_handler`: Extract `State(state)`. Get runtime or return `internal_error`. Create `CoreUseCases::new(runtime.as_ref())`. Call `usecases.get_settings().execute().await`. On Ok, return `Json(json!({ "data": settings, "ts": chrono::Utc::now().timestamp_millis() }))`. On Err, return `internal_error(e)`.

- `update_settings_handler`: Extract `State(state)` and `Json(settings): Json<uc_core::settings::Settings>` (using `Result<Json<Settings>, JsonRejection>` pattern for error handling). If body parse fails, return 400 with `{"error": "invalid_settings", "message": "malformed settings JSON"}`. Get runtime, create CoreUseCases, call `usecases.update_settings().execute(settings).await`. On success return `Json(json!({"success": true}))`. On error return `internal_error(e)`.

- IMPORTANT: Do NOT add any OS-level side effects (autostart, keyboard shortcuts, device name). Only call `update_settings().execute()`.

**Create `src-tauri/crates/uc-daemon/src/api/encryption.rs`:**

```rust
pub fn router() -> Router<DaemonApiState> {
    Router::new()
        .route(http_route::ENCRYPTION_STATE, get(get_encryption_state_handler))
        .route(http_route::ENCRYPTION_UNLOCK, post(unlock_handler))
        .route(http_route::ENCRYPTION_LOCK, post(lock_handler))
}
```

- `get_encryption_state_handler`: Extract `State(state)`. Get runtime. Call `runtime.encryption_state().await` and `runtime.is_encryption_ready().await`. Map to wire format:
  - `Uninitialized` -> `{ "state": "uninitialized", "has_keyslot": false, "requires_passphrase": false }`
  - `Initialized` + is_ready=true -> `{ "state": "unlocked", "has_keyslot": true, "requires_passphrase": false }`
  - `Initialized` + is_ready=false -> `{ "state": "locked", "has_keyslot": true, "requires_passphrase": true }`
  - `Initializing` -> same as locked (transient state)

- Define request DTO (NO Debug derive to prevent passphrase logging):

  ```rust
  #[derive(Deserialize)]
  struct UnlockRequest {
      passphrase: String,
  }
  ```

- `unlock_handler`: Extract `State(state)` and `Json(body): Json<UnlockRequest>`. Get runtime. Create `CoreUseCases`. Call `usecases.unlock_encryption_with_passphrase().execute(Passphrase(body.passphrase)).await`. On success:
  1. Broadcast WS event: `state.event_tx.send(DaemonWsEvent { topic: ws_topic::ENCRYPTION.to_string(), event_type: ws_event::ENCRYPTION_SESSION_READY.to_string(), session_id: None, ts: chrono::Utc::now().timestamp_millis(), payload: json!({}), })` (ignore send error -- no subscribers is ok).
  2. Return `Json(json!({"success": true}))`.
     On error: match `UnlockWithPassphraseError`:
  - `NotInitialized` -> 400 with `{"error": "not_initialized", "message": "encryption is not initialized"}`
  - `UnwrapFailed(_)` -> 401 with `{"error": "wrong_passphrase", "message": "incorrect passphrase"}`
  - Other errors -> `internal_error`

- `lock_handler`: Extract `State(state)`. Get runtime. Call `runtime.deps.security.encryption_session.clear().await`. On success return `Json(json!({"success": true}))`. On error return `internal_error`.

  Note: `lock_handler` accesses `encryption_session` directly from `runtime.deps.security.encryption_session` (the `clear()` method on `EncryptionSessionPort`). There is no lock use case -- it's a single port call.
