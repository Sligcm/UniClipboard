# Plan 82-03: Remove uc:// protocol handler and dead code cleanup

## Goal

Remove the uc:// custom protocol handler from main.rs and the protocol.rs module from uc-tauri, now that all blob/thumbnail serving is handled by daemon HTTP endpoints and the frontend uses `daemonClient.blobUrl()`.

## Tasks

<task id="82-03-T1">
  <name>Remove uc:// protocol handler and CORS helpers from main.rs</name>
  <read_first>
    - src-tauri/src/main.rs
    - src-tauri/crates/uc-tauri/src/protocol.rs
  </read_first>
  <action>
    In `src-tauri/src/main.rs`:

    1. **Remove the `.register_asynchronous_uri_scheme_protocol("uc", ...)` call** (lines ~347-353) and its surrounding comment block (lines ~354-358 about manual verification). The builder chain should go directly from the preceding `.on_page_load(...)` handler to `.plugin(tauri_plugin_notification::init())`.

    2. **Remove the following functions entirely** (they are only used by the uc:// handler):
       - `fn is_allowed_cors_origin(origin: &str) -> bool` (~lines 39-46)
       - `fn set_cors_headers(response: &mut Response<Vec<u8>>, origin: Option<&str>)` (~lines 48-70)
       - `fn build_response(status, content_type, body, origin) -> Response<Vec<u8>>` (~lines 72-95)
       - `fn text_response(status, message, origin) -> Response<Vec<u8>>` (~lines 97-104)
       - `async fn resolve_uc_request(app_handle, request) -> Response<Vec<u8>>` (~lines 106-137)
       - `async fn resolve_uc_blob_request(app_handle, blob_id, origin) -> Response<Vec<u8>>` (~lines 139-180)
       - `async fn resolve_uc_thumbnail_request(app_handle, representation_id, origin) -> Response<Vec<u8>>` (~lines 182-226)

    3. **Remove unused imports** that were only needed for the uc:// handler:
       - `use tauri::http::{header::ACCESS_CONTROL_ALLOW_METHODS, header::ACCESS_CONTROL_ALLOW_ORIGIN, header::CONTENT_TYPE, header::HeaderValue, Response, StatusCode};` — remove the specific headers that are no longer used. Keep any imports still used elsewhere in main.rs.
       - `use uc_tauri::protocol::{parse_uc_request, UcRoute};` — remove entirely.

    4. **Keep all other code in main.rs untouched** — daemon bootstrap, setup, plugin registration, command handlers, etc.

  </action>
  <acceptance_criteria>
    - `main.rs` does NOT contain `register_asynchronous_uri_scheme_protocol`
    - `main.rs` does NOT contain `fn resolve_uc_request`
    - `main.rs` does NOT contain `fn resolve_uc_blob_request`
    - `main.rs` does NOT contain `fn resolve_uc_thumbnail_request`
    - `main.rs` does NOT contain `fn is_allowed_cors_origin` (the main.rs version; daemon has its own)
    - `main.rs` does NOT contain `fn set_cors_headers`
    - `main.rs` does NOT contain `fn build_response`
    - `main.rs` does NOT contain `fn text_response`
    - `main.rs` does NOT contain `parse_uc_request`
    - `main.rs` does NOT contain `UcRoute`
    - `cd src-tauri && cargo check` passes
  </acceptance_criteria>
</task>
