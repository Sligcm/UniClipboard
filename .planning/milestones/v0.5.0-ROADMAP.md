# Milestone v0.5.0: Local Encrypted Search

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 88-93 (+ inserted Phase 92.1)
**Total Plans:** 14 planned items, plus manual Phase 93 completion backfilled at archive time

## Overview

v0.5.0 added local encrypted clipboard search on top of the existing sync product. Search terms are protected on disk, the daemon exposes query/status/rebuild flows, the CLI can query and rebuild the index, QuickPanel uses backend-backed keyword search, and Dashboard exposes search with content-type and time-range controls.

## Milestone Summary

**Decimal Phases:**

- Phase 92.1: CLI Search Commands (inserted after Phase 92 to expose the new search surface outside the UI)

**Key Decisions:**

- Use HKDF-derived search keys and HMAC-tagged terms so local search never stores plaintext tokens on disk.
- Use a version-flag rebuild strategy instead of table renames so rebuilds do not depend on exclusive-lock cutovers.
- Reuse existing clipboard row rendering in search surfaces instead of building a parallel search-only result UI.

**Issues Resolved:**

- Local encrypted search now exists across backend, CLI, QuickPanel, and Dashboard.
- Delete flows now remove search records in sync with clipboard history changes.
- Search rebuilds can be triggered and observed without taking the product offline.

**Issues Deferred:**

- No standalone milestone audit file was produced before archive.
- Phase 93 planning records were not created during execution and were backfilled only at archive time.

**Technical Debt Incurred:**
