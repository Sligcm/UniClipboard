<objective>
Create the four search use cases in `uc-app/src/usecases/search/` as thin orchestrators over `SearchIndexPort`, each in its own file with unit tests that exercise a mock port. Wire the new submodule into `usecases/mod.rs` so downstream consumers can use the types through `crate::usecases::{IndexClipboardEntry, RemoveIndexedEntry, SearchClipboardEntries, RebuildSearchIndex}`.

Purpose: SIDX-01 requires that clipboard entries can be indexed on capture. The capture handler (Phase 92) cannot call the port directly — the use case layer must exist so capture, rebuild, search-query, and manual-remove paths all go through a stable application boundary that is mockable and tracing-instrumented. This plan produces the entire use case surface for all four search operations in a single vertical slice because they share identical port-delegation structure, imports, and test scaffolding.

Output: Five new Rust files in `uc-app/src/usecases/search/` and one modified `usecases/mod.rs`. All four use cases compile, export publicly, and have passing unit tests against a stub `SearchIndexPort` mock.
</objective>
