<objective>
Implement the Phase 90 search pipeline in `uc-infra`: text extraction, deterministic tokenization, HKDF-based search key derivation, HMAC tag generation, and a pipeline helper that emits `SearchDocument` plus `Vec<SearchPosting>` ready for the future SQLite adapter. This plan stops at pure helpers and tests; it does not implement live query SQL or `SqliteSearchIndex`.

Purpose: SIDX-03 through SIDX-06 are all about turning representative clipboard content into profile-scoped encrypted index terms without leaking plaintext tokens or reusing raw `MasterKey` bytes. This plan creates the reusable helpers that Phase 91 and Phase 92 will plug into.

Output: dependency additions in `uc-infra/Cargo.toml`, four new `uc-infra::search` modules, deterministic extractor/tokenizer/derivation tests, and a single pipeline entry point for building search documents and postings.
</objective>

    - `pub fn new(encryption_session: Arc<dyn EncryptionSessionPort>, key_scope: Arc<dyn KeyScopePort>) -> Self`
    - `impl SearchKeyDerivationPort for HkdfSearchKeyDerivation`

    Implement `derive_search_key()` as follows:
    1. call `self.encryption_session.get_master_key().await`
    2. map any missing/unready condition to `SearchError::SessionLocked`
    3. call `self.key_scope.current_scope().await`
    4. run `Hkdf::<Sha256>::new(Some(scope.profile_id.as_bytes()), master_key.as_bytes())`
    5. `expand(b"uniclipboard-search-index/v1", &mut [0u8; 32])`
    6. wrap the result with `SearchKey::from_bytes`

    Also add a private helper:
    - `pub(crate) fn term_tag(search_key: &SearchKey, normalized_token: &str) -> anyhow::Result<Vec<u8>>`
      - instantiate `Hmac<Sha256>` from `search_key.as_bytes()`
      - update with `normalized_token.as_bytes()`
      - return the 32-byte tag as `Vec<u8>`

    Add unit tests that prove:
    - same master key + same profile => same `SearchKey`
    - same master key + different profile => different `SearchKey`
    - missing master key => `SearchError::SessionLocked`
    - `term_tag()` returns 32 bytes
    - no helper accepts `MasterKey` directly as the HMAC key argument

    Do not add any `blake3::derive_key` or `blake3::keyed_hash` call in this file.
