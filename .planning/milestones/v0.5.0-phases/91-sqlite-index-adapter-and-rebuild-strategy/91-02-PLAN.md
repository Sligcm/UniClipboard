<objective>
Add the truthful rebuild half of `SqliteSearchIndex`: temp-table rebuild workspace, in-memory mirror state, blocked-for-the-full-window behavior, transactional copy-in cutover, failure persistence, and concurrency tests that prove the adapter avoids the `RENAME TABLE` lock trap while preserving mid-rebuild inserts and deletes. This plan also makes `rebuild()` the stable adapter entrypoint that Phase 92 can call for both manual rebuilds and first-unlock auto-backfill.

Purpose: Phase 91 is only complete when the adapter can rebuild safely under real SQLite conditions, not just when it can answer a live query. The rebuild path is where stale results, deleted-entry resurrection, and WAL lock assumptions can all fail if the plan is shallow.

Output: rebuild temp-table coordination in `sqlite_index.rs`, reusable rebuild test support in `test_support.rs`, and real temp-file SQLite tests for blocked-state, mid-rebuild mirroring, delete consistency, and concurrent-read cutover.
</objective>

    Extend `src-tauri/crates/uc-infra/src/search/sqlite_index.rs` with one authoritative rebuild coordinator.

    Add these concrete types and helpers:

    - `struct ActiveRebuild { profile_id: String, temp_document_table: String, temp_posting_table: String, target_version: String }`
    - `rebuild_state: Arc<tokio::sync::RwLock<Option<ActiveRebuild>>>` field on `SqliteSearchIndex`
    - `fn create_rebuild_tables(conn: &mut SqliteConnection, state: &ActiveRebuild) -> Result<(), SearchError>`
    - `fn drop_rebuild_tables(conn: &mut SqliteConnection, state: &ActiveRebuild) -> Result<(), SearchError>`
    - `fn insert_temp_entry(conn: &mut SqliteConnection, state: &ActiveRebuild, document: &SearchDocument, postings: &[SearchPosting]) -> Result<(), SearchError>`
    - `fn delete_temp_entry(conn: &mut SqliteConnection, state: &ActiveRebuild, entry_id: &EntryId) -> Result<(), SearchError>`
    - `fn finalize_rebuild(conn: &mut SqliteConnection, state: &ActiveRebuild, profile_id: &str, completed_at_ms: i64) -> Result<(), SearchError>`

    Use this exact temp-table naming prefix:

    - document table: `tmp_search_document_rebuild_`
    - posting table: `tmp_search_posting_rebuild_`

    `rebuild()` must execute in this exact sequence:

    1. resolve `profile_id`
    2. ensure/load meta row
    3. update `search_index_meta.search_blocked = true` and `last_rebuild_started_at_ms = now`
    4. emit `RebuildProgress { stage: RebuildStage::Started, indexed: 0, total: entries.len() as u32 }`
    5. create one `ActiveRebuild` using `CURRENT_INDEX_VERSION.to_string()` as `target_version`
    6. create the temp tables with the same columns used by the active search tables
    7. store `ActiveRebuild` inside `self.rebuild_state`
    8. write every supplied `(SearchDocument, Vec<SearchPosting>)` pair into the temp tables, emitting `RebuildStage::Indexing` after each batch of 100 entries or after the final partial batch
    9. call `finalize_rebuild()` inside one write transaction that:
       - deletes active `search_posting` rows for `profile_id`
       - deletes active `search_document` rows for `profile_id`
       - inserts all temp posting rows into `search_posting`
       - inserts all temp document rows into `search_document`
       - updates `search_index_meta.index_version = CURRENT_INDEX_VERSION`
       - updates `search_index_meta.search_blocked = false`
       - updates `search_index_meta.last_rebuild_completed_at_ms = completed_at_ms`
    10. emit `RebuildProgress { stage: RebuildStage::Complete, indexed: entries.len() as u32, total: entries.len() as u32 }`
    11. drop temp tables and clear `self.rebuild_state`

    Error policy must be concrete:

    - do not use `RENAME TABLE`
    - on any error, emit `RebuildStage::Failed`
    - clear `self.rebuild_state`
    - drop temp tables best-effort
    - leave `search_blocked = true`
    - return `SearchError::Internal(...)`

    Create `src-tauri/crates/uc-infra/src/search/test_support.rs` with reusable helpers for this phase:

    - temp-file DB fixture that returns `(DbPool, SqliteSearchIndex, db_path)`
    - fixed profile-scope and fixed search-key-derivation stubs
    - `hold_read_transaction(db_path: &Path) -> ReadTxnHandle` that opens a second SQLite connection, begins a read transaction, and keeps it alive until dropped

    Add these tests:

    - `rebuild_cutover_sets_blocked_then_clears_on_success`
    - `rebuild_failure_leaves_meta_blocked`
    - `rebuild_does_not_use_rename_table`

    Use a test-only fault injection hook if needed to force the failure path, but keep that hook behind `#[cfg(test)]` and local to `sqlite_index.rs`.
