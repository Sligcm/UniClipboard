    Create `src-tauri/crates/uc-daemon/src/api/dto/search.rs` with these exact transport shapes:

    - `SearchResultDto` mirroring `SearchResult`
    - `SearchQueryResponse { data: Vec<SearchResultDto>, total: u32, has_more: bool, ts: i64 }`
    - `SearchStatusData { state: String, reason: Option<String>, last_rebuild_started_at_ms: Option<i64>, last_rebuild_completed_at_ms: Option<i64> }`
    - `SearchStatusResponse { data: SearchStatusData, ts: i64 }`
    - `SearchRebuildAcceptedResponse { data: SearchRebuildAcceptedData, ts: i64 }`
    - `SearchRebuildAcceptedData { accepted: bool }`

    Use `#[serde(rename_all = "camelCase")]` on every response struct.

    Create `src-tauri/crates/uc-daemon/src/api/search.rs` with `pub fn router() -> Router<DaemonApiState>` that mounts:

    - `GET http_route::SEARCH_QUERY`
    - `GET http_route::SEARCH_STATUS`
    - `POST http_route::SEARCH_REBUILD`

    Implement the query parser with these exact params and defaults:

    - `query` required
    - `operator` optional (`and` or `or`)
    - `timePreset` optional
    - `fromMs` + `toMs` optional pair
    - repeated `fileTypes`
    - repeated `extensions`
    - `limit` default `50`, clamped to `200`
    - `offset` default `0`

    Parser rules must be concrete:

    - `operator=and|or` maps directly to `QueryOperator`
    - if `operator` is absent, infer from raw `query`
    - mixed standalone `AND` and `OR` tokens return `ApiError { status: 400, code: "invalid_query", message: "mixed AND/OR operators are not supported" }`
    - invalid `timePreset`, invalid `fileTypes`, or mismatched `fromMs` / `toMs` return `bad_request`
    - strip standalone operator tokens before passing `query_string` to the use case

    Lock guard rules:

    - before `/search/query`, `/search/status`, and `/search/rebuild`, call `runtime.is_encryption_ready().await`
    - if false, return `ApiError { status: StatusCode::LOCKED, code: "session_locked".to_string(), message: "encryption session is locked".to_string() }`

    Route behavior:

    - `/search/query` delegates through `CoreUseCases::new(runtime.as_ref()).search_clipboard_entries().execute(query).await`
    - `/search/status` reads `SearchCoordinator` snapshot and returns `200` payload for unlocked sessions
    - `/search/rebuild` asks the coordinator for a manual rebuild, returns `202` on accept, `409` with code `rebuild_already_running` when the coordinator rejects a concurrent request

    Register the new router in `src-tauri/crates/uc-daemon/src/api/routes.rs` with `.merge(crate::api::search::router())` inside `router_l2_plus`.

    Register the new DTOs and route module in `src-tauri/crates/uc-daemon/src/api/openapi.rs`.

    Do not duplicate search execution or rebuild logic in the route module. Parsing and HTTP/status mapping only.
