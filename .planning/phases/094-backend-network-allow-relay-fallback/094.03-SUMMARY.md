## Decisions Made

- **placeholder 默认值与 `core::NetworkSettings::default()` 保持一致（true）：** 占位期 webserver 返回的 `network.allow_relay_fallback` 与 plan 02 完成后真实读取的语义没有差异（核心 default 即 true），避免误导客户端的临时值。这是 Pitfall 2 防御兜底 — 默认值传统是 true（允许 fallback / 老用户行为不变）。
- **`restart_required` 占位 = false：** 真实计算 `payload.network.is_some()` 在 plan 04 落地。占位 false 不会误导现有客户端：当前没有客户端发送 `network` patch 字段（plan 04 wire-up 之前），任何 PUT 都不会真正改变 network 设置，"无需重启"语义正确。
- **未在本 plan 修改 `settings_patch_from_dto`：** 该函数包含 patch DTO → application Patch 的映射，是 plan 04 范围的核心改动；本 plan 只做必要的"让编译通过"的最小修复。
