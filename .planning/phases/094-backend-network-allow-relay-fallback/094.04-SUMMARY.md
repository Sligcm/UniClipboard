## Decisions Made

- **暴露可见性选 `pub #[doc(hidden)]` 而非 `pub(crate)`：** Integration test 在 `tests/` 子目录是外部 crate（cargo test 把每个 .rs 当独立 binary，链接 uc-webserver lib），`pub(crate)` 不够。`#[doc(hidden)]` 让 rustdoc 不展示，语义上仍是 crate-internal helper。
- **取消 `tempfile::TempDir` 句柄：** PLAN.md 范围声明里写"fixture 用 in-memory SettingsPort + tempdir 持久化点"，但实际 InMemorySettings 不读写文件，tempdir 是占位证据。删除句柄 + 不引入 tempfile dev-dep，理由：(1) 减少 dev-dep；(2) Mutex<Settings> 已等价承担"写盘 / 读取一致"语义，与 FileSettingsRepository 行为同形；(3) 与 uc-application/src/facade/settings/facade.rs:55-84 既有 `InMemorySettings` 模式严格对齐
- **case 5 显式枚举 GeneralSettingsPatchDto 全字段为 None：** plan 03 task 1 只给 `SettingsPatchDto` + `NetworkSettingsPatchDto` 加 derive(Default)（BLOCKER 5），`GeneralSettingsPatchDto` 没 derive(Default)。`..Default::default()` 不可用，只能显式枚举（auto_start = Some(true) 为 driver 字段，其它字段 None）。接受这一 brittleness：未来若 GeneralSettingsPatchDto 加字段可能引入编译错误，但与 plan 03 决策一致
- **不写额外的 `#[cfg(test)] mod network_mapping_tests` lib 单元测试：** PLAN.md task 1 行为段建议在 settings.rs 末尾加 7 个单元测试（2 mapping + 5-case truth-table），但用户简化指令明确只要求 task 1 替换 placeholder + cargo check + commit。truth-table 5 cases 已在 task 2 的 `restart_required_truth_table` integration test 里完整覆盖（同源逻辑 — `payload.network.is_some()`），mapping 透明搬运 2 case 已在 `roundtrip_network_disable` integration test 里覆盖（PUT false + GET false 即同时验证 patch_from_dto.network 段 + view_to_dto.network 段 + apply_settings_patch.network 段）。功能等价 — 用户指令优先于 plan action 文本
