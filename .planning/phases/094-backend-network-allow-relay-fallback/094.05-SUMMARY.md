## Decisions Made

### D1: D-A1 落地 — helper 物理位置 = network_policy.rs

按 plan 与 CONTEXT.md `<decisions>` D-A1 锁定：新模块 `uc-bootstrap/src/network_policy.rs`，`pub(crate)`（不对外暴露），单测就近放在同文件 `#[cfg(test)] mod tests`。

### D2: D-B1 选项 B 现状决策（接受隐式契约）

按 CONTEXT.md `<decisions>` D-B1（修订版）+ `<deferred>` 锁定：

- `SettingsPort::load` 当前签名 `async fn load(&self) -> anyhow::Result<Settings>` 不区分 NotFound vs Parse
- `FileSettingsRepository::load` (`repository.rs:166-168`) 已对 NotFound 兜底返回 `Settings::default()`，所以 builders 层只能看到 Parse/IO 错误
- 因此 `wired.deps.settings.load().await.map_err(...)?` 硬失败语义与 D-B1 期望的"NotFound 静默走 default"等价（NotFound 已在 infra 层吃掉）
- 这违反 `uc-infra/AGENTS.md §4.2`（实现细节不能泄漏到上层）—— 有意识接受为临时技术债务，CONTEXT.md `<deferred>` 已记录"v0.7.x 后续 phase 必须偿还（实施 SettingsLoadError discriminated enum）"

### D3: D-B3 启动日志格式定稿

```rust
tracing::info!(
    target: "settings.network",
    allow_relay_fallback,
    disable_relays = iroh_config.disable_relays,
    "applying network.allow_relay_fallback={} → disable_relays={}",
    allow_relay_fallback,
    iroh_config.disable_relays,
);
```

字段名（snake_case rust）= `allow_relay_fallback` / `disable_relays`，与代码字段一致；message 包含 → 箭头便于 support 排障 ctrl-F；`disable_relays` 字段值取自 `iroh_config.disable_relays`（helper 输出），不在 OTLP 加 attribute（Pitfall 6 不联动）。
