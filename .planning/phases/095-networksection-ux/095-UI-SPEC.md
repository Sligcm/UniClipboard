## Spacing Scale

Project uses Tailwind default 4px-based scale (declared `--spacing: 0.25rem`). Phase 95 declared values — all multiples of 4 (Tailwind tokens in parentheses):

| Token | Value | Tailwind | Usage in Phase 95 |
|-------|-------|----------|-------------------|
| xs | 4px  | `gap-1` / `space-y-1` | Icon ↔ label gap, vertical text rhythm inside Popover items |
| sm | 8px  | `gap-2` / `space-y-2` / `p-2` | RestartBanner icon ↔ text gap, button group gap, label ↔ info-icon gap |
| md | 12px | `px-3` / `py-3` | Popover internal padding (existing `p-3` from `popover.tsx:19`); RestartBanner internal padding |
| md-lg | 16px | `px-4` / `gap-4` | SettingRow horizontal padding (existing `px-4 py-3` from `SettingRow.tsx:20`) |
| lg | 24px | `mt-6` between SettingGroups (matches `SyncSection.tsx:237`) |
| xl | 32px | reserved for future Sections — not used in Phase 95 |

Exceptions:
- **Popover content width:** fixed `w-72` (288px) inherited from existing `popover.tsx:19` — do NOT override; 4 disclosure items fit comfortably at this width.
- **Info icon hit-target:** lucide `Info` icon is rendered at `size-3.5` (14px), wrapped in a `<button>` with `p-1` (4px padding all around) to reach a minimum 22×22px hit area. Wrapper button is `inline-flex items-center justify-center rounded-md text-muted-foreground hover:text-foreground hover:bg-accent`.
- **RestartBanner top margin inside SettingGroup:** Banner sits **above** the LAN-only Switch row, separated by the existing `divide-y divide-border/40` (1px) from `SettingGroup.tsx:18`. No extra `mt-*` — the divider is the separator.

## Color

Project uses `oklch()` semantic tokens via CSS variables in both light (`:root`) and dark (`.dark`) modes (`globals.css:50-119, 422-454`). Phase 95 maps to existing tokens — **no new colors introduced**.

| Role | Token | oklch (light / dark) | Usage in Phase 95 |

**Accent reserved for (explicit list — not "all interactive elements"):**
1. LAN-only `<Switch>` checked-state track (existing primitive)
2. "立即重启" `<Button variant="default">` primary fill (the single CTA in this phase)
3. Focus ring on info-icon Popover trigger button (existing focus-visible token)
4. Focus ring on Switch (existing primitive)

**Accent NOT used for:** Popover items, RestartBanner background fill, info-icon idle color, descriptive text, or any decorative purpose.

**Dark mode:** All color choices are token-based; visually verify in dark mode that (a) RestartBanner `bg-accent/40` is distinguishable from `bg-card` surface (dark accent is `0.371 0 0` — sufficient contrast against `0.205 0 0` card); (b) Switch checked thumb (`bg-primary-foreground` in dark) is visible against primary track (`0.87 0 0`).

## Layout Contract

```
NetworkSection (renders SettingGroup)
└── SettingGroup (title = "网络设置" / "Network", existing card surface)
    ├── RestartBanner (visible only when pending)
    │   ├── [icon: RefreshCw, text-foreground, size-4, mt-0.5]
    │   ├── [text column, gap-y-1]
    │   │   ├── message (text-sm, text-foreground)
    │   │   └── error?  (text-xs, text-destructive)  // only on B3 / PUT fail
    │   └── [action column, ml-auto, gap-x-2]
    │       ├── Button "立即重启" (variant="default", size="sm")  // hidden when error visible? NO — keep visible; switch to "Retry" only when explicitly error
    │       └── (error case) Button "重试" (variant="outline", size="sm") + Button X (variant="ghost", size="icon")
    │   ├ Container: rounded-none (top of group, shares group's outer radius)
    │   ├ bg-accent/40, border-b border-border/40 (matches divide token)
    │   └ px-4 py-3 (matches SettingRow rhythm)
    └── SettingRow
        ├── label = "LAN-only 模式" / "LAN-only Mode"
        ├── labelExtra = LanOnlyDisclosure (info-icon button, ml-1)
        ├── description = (full sentence with "重启后生效" suffix)
        └── children = <Switch checked={!allowRelayFallback} ...>
```

**Banner placement (D-A2 enforcement):** RestartBanner is rendered as the **first child** inside `SettingGroup`'s inner div (the `divide-y` container at `SettingGroup.tsx:18`). The existing `divide-y divide-border/40` automatically draws the 1px separator between Banner and Switch row — no extra margin/border needed.

**Three-state visual (D-A3 enforcement):**
- `applied OFF` (default): no Banner, Switch off
- `applied ON`: no Banner, Switch on
- `pending change` (either direction): Banner visible, Switch reflects the user's pending choice
- **Switch styling never changes between states** — pending signal is carried entirely by Banner's appear/disappear

**Popover anchor:** `PopoverTrigger` wraps the info-icon button; `PopoverContent` uses `align="start" sideOffset={8}` (existing default). Width inherits `w-72` from primitive.

**Mobile / narrow-window behavior:** Settings window is desktop-only (Tauri); minimum width is the existing Settings page constraint. Banner action column wraps below message at very narrow widths via `flex flex-wrap`.

## Interaction Contract

| Trigger | Behavior | Token / Source |
|---------|----------|----------------|
| User clicks Switch | (1) optimistic local state update; (2) Banner appears immediately; (3) `useDebounce` waits 500ms; (4) PUT `/settings` fires once with last value | D-D2, D-D3, ROADMAP Pitfall 10 |
| User clicks "立即重启" Button | (1) button enters `loading` state, label switches to `restartingButton` zh "正在重启…"; (2) `invokeWithTrace('restart_app')`; (3) on success: process exits — no UI feedback needed; (4) on failure: button returns to idle, error text appears under message, "重试" button appears next to "立即重启" | D-B2, D-B3 |
| User clicks "重试" Button | Same as "立即重启" click — re-invoke command | D-B3 |
| User clicks error-dismiss `X` | Clears local `error` state — Banner main message + "立即重启" Button stay (pending state still real) | New (Discretion: error is dismissable; pending notice is not) |
| User hovers info-icon | NO action — Popover is **click-only** per D-C1 | D-C1 |
| User clicks info-icon button | Popover opens; click-outside or Esc closes; content is selectable & copyable | D-C1, Radix default |
| User clicks Switch when PUT fails | Switch UI rolls back to previous value, Banner does not appear, inline `saveError` text renders below SettingRow for ~5 seconds (auto-dismiss via `setTimeout`) then disappears | Discretion (planner direction): Switch回滚 + RestartBanner不出现 + 显示inline error |
| Component mount | Call `get_restart_state` Tauri command once; if `settings_mtime > process_started_at` AND `setting.network.allowRelayFallback` differs from boot-time bind value (defer "differs" check to Phase 96; Phase 95 simple rule: `mtime > process_started_at` ⇒ pending) → set `pending=true`, render Banner | D-D1 |
