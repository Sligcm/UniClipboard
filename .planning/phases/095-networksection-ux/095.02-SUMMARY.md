## Decisions Made

- **PROCESS_STARTED_AT 放 builder 链外独立语句** —— PLAN 建议放在 builder 链外或链内 setup 早期。最终选放在链外、`tauri::Builder::default()` 上方独立语句执行：
  - 优点 1: OnceLock::set 不依赖 app handle，无需进入 `.setup(|app| ...)` 闭包
  - 优点 2: 在第一次 invoke_handler 之前 set —— 避免任何竞态
  - 优点 3: 本地变量作用域短，对 builder 链可读性零影响
- **Test 5 fence test 用 const fn 编译期断言** —— 初版使用 `assert!(true, "...")` 触发 clippy::assertions_on_constants 警告（Rule 1 bug，自我引入）。修复方案：用 `const _ASSERT_ONCE_LOCK_SYSTEM_TIME: fn() -> &'static OnceLock<SystemTime> = || &PROCESS_STARTED_AT;` 编译期断言 PROCESS_STARTED_AT 类型签名 + `assert_eq!(fence_marker, "fence")` 留 PR diff 关键字。这同时强化了 fence 语义（编译期断言可防误改类型）。
- **Test 4 (`read_process_started_at_returns_non_negative`) 仅做单调非负断言** —— 测试不分 set / not-set 情况。原因：所有测试共享同一进程内存空间，PROCESS_STARTED_AT 是全局 OnceLock，被任何先跑的测试 set 后无法重置；只验证 helper 永远返回非负即可。
