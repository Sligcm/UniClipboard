## Decisions Made

1. **REFACTOR fence 测试块在 RED commit 一次写入** — Plan Task 3 列出的 4 个 fence 测试在测试套件设计时即纳入（与行为测试同套件），让 GREEN commit 后 18/18 一次性全绿；REFACTOR commit 仅追加 NetworkSection.tsx 内联 FENCE 注释 + Pitfall doc 段。这是显式 plan-aware 偏离 — 测试覆盖范围 / fence 守护严格性与 plan 完全一致；commit message 标注 `refactor(95-06)` 满足 acceptance prefix 要求。
2. **saveError 文案在前端构造完整字符串后塞入 state** — `setSaveError(t('...saveError', { message }))` 一次解析；RestartBanner / SettingRow 渲染时只读 `saveError` string，不再二次 t()。i18n key `{{message}}` placeholder 形态与 Plan 03 一致。
3. **Switch onCheckedChange 主动清 restartError + saveError** — 用户重新切 Switch 等于重新声明 pending 意图，旧错误应一并消失。Banner pending 主信息保留（pending 状态仍真实）。
4. **Effect 3 含 `debouncedAllowRelay === persistedAllowRelay` 早返回** — 防止用户切走又切回原值时仍发一次 noop PUT。日常 LAN-only 切换是 boolean 反复横跳的高频场景，这一保护让 daemon /settings 写盘最小化。
5. **isPristineRef 用 useRef 而非 useState** — 仅作为副作用 gate flag，不需要触发 rerender；useRef 避免无意义的 commit phase。
6. **mount 时 get_restart_state 失败仅 log.error 不显式 setError** — 静默失败，组件仍可工作（Banner 默认不可见）；Tauri 进程异常下不阻塞 Switch 操作。
7. **Effect 3 eslint-disable react-hooks/exhaustive-deps** — 故意只监听 `debouncedAllowRelay`；persistedAllowRelay / setting / updateNetworkSetting / t 都是稳定引用或闭包内的派生值，加入会导致每次 setting 刷新都重新 schedule effect 触发不必要的 noop PUT。
