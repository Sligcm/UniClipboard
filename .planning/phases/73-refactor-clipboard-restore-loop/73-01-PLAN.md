CRITICAL TTL NOTE: The existing codebase uses TWO different TTLs:

- Local restore/copy: `Duration::from_secs(2)` (in restore_clipboard_selection.rs, copy_file_to_clipboard.rs)
- Remote push (inbound sync): `REMOTE_SNAPSHOT_HASH_TTL_MS = 60_000` (60 seconds, in sync_inbound.rs)

The coordinator MUST use per-intent TTL to preserve existing behavior:

- LocalRestore / LocalCapture: 2 seconds
- RemotePush: 60 seconds

CRITICAL REMOTE PUSH NOTE: The inbound sync path has an ADDITIONAL step after successful write:
`self.clipboard_change_origin.set_next_origin(RemotePush, Duration::from_millis(60_000))` — this guards against OS re-encoding loopback. The coordinator's RemotePush path MUST include this `set_next_origin` call after successful write.
