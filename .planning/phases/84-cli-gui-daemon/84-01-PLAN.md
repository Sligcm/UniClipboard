**Current problematic code:**

```rust
// Parse "Session <token>" prefix
let token = auth_value
    .strip_prefix("Session ")
    .unwrap_or(auth_value.as_str())  // BUG: falls back to bare bearer token
    .trim();
if token.is_empty() {
    return (StatusCode::UNAUTHORIZED, ...);
}
// Then JWT verify runs on bare token -> decode failure -> 401 "invalid_session_token"
// The error message is misleading - it's not "invalid", it's "wrong scheme"
```

**Replace with explicit rejection:**

```rust
// Parse "Session <token>" prefix — bare bearer tokens are explicitly rejected.
let Some(token) = auth_value.strip_prefix("Session ") else {
    return (
        StatusCode::UNAUTHORIZED,
        axum::Json(serde_json::json!({
            "error": "invalid_auth_scheme",
            "message": "expected 'Session ' prefix; use POST /auth/connect to obtain a session token"
        })),
    )
        .into_response();
};
let token = token.trim();
if token.is_empty() {
    return (
        StatusCode::UNAUTHORIZED,
        axum::Json(serde_json::json!({
            "error": "missing_session_token"
        })),
    )
        .into_response();
}
```

**What changed:**

1. `unwrap_or(auth_value.as_str())` replaced with explicit `else` branch returning 401
2. New error code: `"invalid_auth_scheme"` (not `"invalid_session_token"`) so clients can distinguish bare bearer attempts from actual JWT failures
3. Error message includes hint: "use POST /auth/connect to obtain a session token"
4. The `unwrap_or` + `.trim()` + `if token.is_empty()` chain simplified to `strip_prefix` + `else` + `trim` + `if empty` (the empty check still handles `Authorization: Session ` with no token value)

**Why this matters (D-04):** Explicit rejection means CLI's current bare bearer pattern gets a clear error ("invalid_auth_scheme") rather than a misleading "invalid_session_token" (JWT decode failure). This makes it obvious the CLI needs to migrate to session exchange.

#[tokio::test]
async fn middleware_rejects_bare_bearer_token_explicitly() {
    // Verify that auth_extractor_middleware returns "invalid_auth_scheme" error
    // when a bare bearer token (no "Session " prefix) is provided.
    use axum::body::Body;
    use axum::http::{Request, StatusCode};
    use tower::ServiceExt;

    let state = crate::api::server::DaemonApiState::new_for_test();
    let app = crate::api::server::build_router_for_test(state.clone());

    // Send request with bare bearer token (no "Session " prefix)
    let response = app
        .oneshot(
            Request::builder()
                .method("GET")
                .uri("/status")
                .header("Authorization", "Bearer my-raw-bearer-token")
                .body(Body::empty())
                .unwrap(),
        )
        .await
        .unwrap();

    assert_eq!(
        response.status(),
        StatusCode::UNAUTHORIZED,
        "bare bearer token should be rejected with 401"
    );

    let body = axum::body::to_bytes(response.into_body(), 4096).await.unwrap();
    let json: serde_json::Value = serde_json::from_slice(&body).unwrap();
    assert_eq!(
        json["error"].as_str(),
        Some("invalid_auth_scheme"),
        "error code should be 'invalid_auth_scheme', got: {:?}",
        json["error"]
    );
}
