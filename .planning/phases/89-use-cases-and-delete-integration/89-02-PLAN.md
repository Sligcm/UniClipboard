<objective>
Extend the existing `DeleteClipboardEntry` use case in `uc-app/src/usecases/delete_clipboard_entry.rs` with an optional `SearchIndexPort` dependency. When present, calling `execute()` synchronously invokes `search_index.remove_entry(entry_id)` as part of the delete chain. Failures are logged as warnings and do not block the delete (D-07), matching the existing file cache cleanup error policy. Injection uses a `.with_search_index(Arc<dyn SearchIndexPort>)` builder method (D-08), consistent with the existing `.with_file_cache_dir()` pattern.

Purpose: SIDX-02 requires that deleting a clipboard entry synchronously removes its search document and all search_posting rows. This keeps the search index consistent with user-visible clipboard history without requiring a separate background job. The builder pattern keeps the change backwards-compatible — runtimes that do not need search (tests, CLI-only scenarios) can continue to construct `DeleteClipboardEntry` without the search port.

Output: Modified `delete_clipboard_entry.rs` with a new optional field, builder method, integration call inside `execute()`, and new unit tests (spy mock for the happy path, and a warn-and-continue test for the error path).
</objective>
