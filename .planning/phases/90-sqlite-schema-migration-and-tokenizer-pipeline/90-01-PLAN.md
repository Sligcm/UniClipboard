<objective>
Add the SQLite foundation for local encrypted search in `uc-infra`: a Diesel migration, Diesel schema declarations, search constants, and adapter-owned row structs that can represent the Phase 90 persistence model without widening `uc-core`. This plan owns the on-disk truth for `search_document`, `search_posting`, and `search_index_meta`, plus the minimal constants needed by the later tokenizer pipeline.

Purpose: SIDX-07 requires versioned search schema support, and Phase 90 requires profile isolation from day one. The safest way to achieve this is to create the search tables and row helpers in `uc-infra`, keep `profile_id` as an infra persistence concern, and expose stable constants for `index_version` and `field_mask` values that later phases can reuse.

Output: a new Diesel migration directory, updated `schema.rs`, a new `uc-infra::search` module with `constants.rs` and `rows.rs`, and migration/row tests that prove the schema exists and maps cleanly to current search-domain contracts.
</objective>
