    ```rust
        pub query: String,
        pub operator: Option<String>,
        pub time_preset: Option<String>,

        pub to_ms: Option<i64>,
        pub file_types: Vec<String>,
        pub extensions: Vec<String>,
        pub limit: u32,
        pub offset: u32,
    }

    pub struct DaemonSearchRequestError {
        pub path: String,
        pub status: reqwest::StatusCode,
        pub code: Option<String>,
        pub message: String,
    }
    ```

    Implement these exact client methods:

    - `pub async fn query(&self, request: SearchQueryRequest) -> anyhow::Result<SearchQueryResponse>`
    - `pub async fn status(&self) -> anyhow::Result<SearchStatusResponse>`
    - `pub async fn rebuild(&self) -> anyhow::Result<SearchRebuildAcceptedResponse>`

    Request rules must be concrete:

    - `query()` uses `http_route::SEARCH_QUERY`
    - `status()` uses `http_route::SEARCH_STATUS`
    - `rebuild()` uses `http_route::SEARCH_REBUILD`
    - query params are encoded in camelCase
    - `fileTypes` and `extensions` are sent as comma-joined strings when non-empty
    - `limit` and `offset` are always sent
    - the client does **not** strip inline `AND` / `OR` or infer operators locally

    Error decoding rules must be concrete:

    - on non-success, read the body once
    - attempt to parse a generic `{ "code": "...", "message": "..." }` payload
    - construct `DaemonSearchRequestError { path, status, code, message }`
    - wrap that structured error with `anyhow!(error)` exactly like the pairing client pattern
