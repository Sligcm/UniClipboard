The Quick Panel is UniClipboard's keyboard-first entry point: a
floating overlay that lists the current space's clipboard history.
Arrow-key your way to an item, hit Enter, and it pastes back into the
app you came from — without ever bringing the main window forward.

## Open and dismiss

| Platform        | Default hotkey   |
| --------------- | ---------------- |
| macOS           | `Cmd + Ctrl + V` |
| Windows / Linux | `Ctrl + Alt + V` |

Press once to open, press again to close. `Esc` also dismisses. The
panel auto-hides when it loses focus — by design it doesn't try to
hold focus indefinitely.

<Callout type="info">
  Hotkey doesn't seem to fire? It's almost always a conflict with another app's shortcut, not a bug.
  Rebind it under [Settings — Shortcuts](../guides/settings#shortcuts) to a free combo. If you'd
  rather not have the panel at all, you can turn the whole feature off under [Settings — Quick
  Panel](../guides/settings#quick-panel) — that unregisters the hotkey immediately.
</Callout>

## Keyboard controls

The panel is meant to be driven entirely by keyboard. The full set:

| Key                      | Action                                        |
| ------------------------ | --------------------------------------------- |
| `↑` / `↓`                | Move selection                                |
| `Enter`                  | Paste the selected item into the previous app |
| Hold `Option` / `Alt`    | Request plain-text paste                      |
| `Esc`                    | Close without pasting                         |
| `Alt + Backspace`        | Remove the selected item from history         |
| `Cmd` / `Ctrl` + `1`–`9` | Jump to items 1–9                             |
| `Cmd` / `Ctrl` + `0`     | Jump to item 10                               |
| Just type                | Live substring filter (case-insensitive)      |
| `:` at start of search   | Switch to advanced search (see below)         |

`Enter` simulates a system paste, so the receiving app sees what looks
like a regular Cmd/Ctrl+V — rich text and files land at the highest
fidelity that the destination app understands.

To paste plain text, hold `Option` / `Alt` while confirming an item:
press `Enter`, click an item, or use a `Cmd` / `Ctrl` + number
shortcut while the modifier is held. UniClipboard writes only the
`text/plain` representation to the system clipboard first; if the item
has no plain-text representation, it silently falls back to the normal
multi-format paste.

## Inline preview

When an item is selected, the right pane renders it in the format
matching its content type:

| Representation | Preview                                                   |
| -------------- | --------------------------------------------------------- |
| Plain text     | Monospace; virtualized scrolling for large text (> 50 KB) |
| Link           | Extracted domain plus the full URL                        |
| Image          | Embedded thumbnail, fit-to-area, with dimensions          |
| Code / HTML    | Same renderer as plain text (no syntax highlighting yet)  |
| File           | List of filenames with icons; multi-file shown row-by-row |

Images larger than 8 MB aren't pulled automatically — the pane shows a
**Load image** button instead, so a big original never blocks the
preview. Click it to fetch the full image on demand.

The preview is **read-only**. To put the item back on the system
clipboard or paste it, return to the list and hit `Enter`.

## Type and time filters

Two filter controls live above the list. They stack — applying both
narrows the result.

- **Type chips**: All / Text / Image / Link / File / Code
- **Time dropdown**: Today, Yesterday, Last 7 days, Last 30 days, This
  week, This month, or All time

Filters are in-memory only and **don't persist** across panel
sessions — every reopen starts at "All". The panel is a use-and-throw
surface, not a workbench.

## Advanced search (`:` mode)

Type `:` at the start of the search box to switch into advanced mode,
where filter conditions go inline with the query:

```text
:type:image 2026
:type:code ext:ts useEffect
:ext:pdf report
```

Supported tokens:

- `type:<text|image|link|file|code>` — restrict by content type
  (equivalent to clicking a type chip).
- `ext:<extension>` — restrict by file extension (only meaningful for
  file payloads).
- Any other token is a substring filter; multiple tokens are AND-ed.

Quoted phrases, boolean operators, and regex are **not** supported —
see [Full-text search — Query syntax](./search#query-syntax).

<Callout type="info">
  The CLI equivalent is `uniclip search` with `--type`, `--ext`, `--from-ms`, and `--to-ms`. See
  [CLI reference — Search](../cli/reference#search).
</Callout>

## Customizing the hotkey

<Tabs items={['Where', 'Combos that tend to be free']}>

<Tab value="Where">

**Settings → Shortcuts → Global → Open Quick Panel**

The recorder accepts a key combo directly — just press it; conflicts
are flagged inline. `Reset` puts it back to the default; `Clear`
unbinds it entirely (after which you can still open the panel from the
tray menu or main window).

</Tab>

<Tab value="Combos that tend to be free">

Combinations that are unlikely to clash with system or IDE shortcuts:

- macOS: `⌃⌥⌘V`, `⌃⌘Space` (clashes with the "previous input source"
  shortcut — disable that first if you want this combo)
- Windows: `Win+V` (clashes with Windows' own clipboard history; turn
  the OS feature off first), `Ctrl+Shift+V`
- Linux: `Super+V`, `Ctrl+Alt+V` (default)

</Tab>

## When to use the panel vs. the main window

| What you want                                      | Best surface                                             |
| -------------------------------------------------- | -------------------------------------------------------- |
| Paste a recent item right now                      | Quick Panel                                              |
| Find something from days ago across devices        | Quick Panel `:` advanced search, or CLI `uniclip search` |
| Long sessions of triage, bulk delete, export       | Main window (Dashboard)                                  |
| Configure filters or per-device content-type rules | [Devices](./devices) / [Settings](../guides/settings)    |

## Troubleshooting

- Hotkey doesn't fire: see
  [Troubleshooting — Clipboard permissions](../help/troubleshooting#clipboard-permissions).
  On macOS confirm **Input Monitoring** is granted and the app was
  fully relaunched afterwards.
- Items you just copied don't appear: the daemon isn't running or the
  clipboard listener was sandboxed. Check `daemon.running` and the
  clipboard logs in `uniclip status`.
- A peer's clip never shows up: the sending device may have it gated
  out — check
  [Devices — Content-type allowlist](./devices#content-type-allowlist)
  on that side.
