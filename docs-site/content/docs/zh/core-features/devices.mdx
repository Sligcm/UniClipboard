设备页是空间成员的实时视图：哪台机器在线、走的是直连还是中继、哪些内容类型被允许从这边发过去。所有动作都对 **当前空间** 生效；切换空间后视图会随空间更新。

## 页面结构

| 区块       | 内容                                                                                                                               |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| 顶部 Hero  | 本机图标 + 设备名 + Peer ID 摘要 + 「切换空间」chip；右侧两个 StatPill 分别显示已配对 / 在线数与同步状态。                         |
| Tab 切换   | 单行两个标签：「**已配对设备**」（桌面 P2P 对端） / 「**手机同步**」（局域网移动客户端）。每个标签自带计数 chip 与右侧的操作按钮。 |
| Tab 内容区 | 当前 tab 的卡片网格（3 列在大屏，小屏自动收为 1–2 列）。最后一格是虚线「添加」占位卡，点开走对应的注册 / 邀请流程。详见下面两节。  |

顶部 Hero 是只读摘要，要改本机名 / 同步开关请去 [设置 → 通用](../guides/settings#通用) 与 [设置 → 同步](../guides/settings#同步)。「切换空间」chip 走的是另一个对话框。

完整的移动端同步配置流程（LAN 监听器、iOS 快捷指令、凭据管理）单独整理在
[移动端同步](./mobile-sync) 这一节。

## 卡片能告诉你什么

每张卡片自上而下：

- **设备图标** —— 根据设备名启发式选色（在线为强调色、离线为灰）；在线时图标右下角再叠一个小绿点。
- **设备名** —— 该设备本机起的名字；过长会被截断。
- **状态行** —— `● 在线` / `○ 离线` ＋（若有）连接地址 mono 显示。
- **通道 chip** —— 卡片底部的传输通道徽标（详见下文）。
- **点击整张卡片** —— 打开居中的「设备设置」modal。

### 传输通道徽标

<Feature since="0.7.0">
  按对端展示 `LAN` / `Direct` / `Relay` 的可达性徽标。底层由 `ConnectionChannelPort`
  单一真相源贯穿到 DTO，GUI、托盘图标与 `uniclip status --json` 三处看到的状态一致。
</Feature>

`Online` 旁的小标签告诉你这次连接走的是哪条路：

| 标签     | 含义                              |
| -------- | --------------------------------- |
| `LAN`    | 通过局域网 mDNS 发现并直连。      |
| `Direct` | NAT 穿透成功，跨网络的 P2P 直连。 |
| `Relay`  | P2P 打洞失败，走加密中继转发。    |

中继看到的永远是密文，参见 [配对与同步 — 中继看到的](../guides/pairing#中继看到的-vs-对端看到的)。**这个标签只反映可达性 / 性能层面**，不影响机密性。

## 添加新设备

在「已配对设备」tab 标签右侧点 `邀请新设备`（卡片网格里最后一张虚线占位卡也是同一个入口）。弹窗会：

1. 调用 sponsor 守护进程生成一段一次性邀请码。
2. 显示邀请码 + 倒计时（默认有效期 5 分钟）。
3. 提供一键复制按钮。

把邀请码送到新设备（同物理位置直接念 / 私聊都行 —— 邀请码不带口令），让对方在新设备上点 **加入已有空间** → 输入邀请码与空间口令。详见 [快速开始 — 添加第二台设备](../getting-started/quick-start#添加第二台设备)。

<Callout type="warn">
  邀请码过期或已被消费就 **无法再用**。这是配对协议的"一次性"性质决定的，不是 bug ——
  重新生成一张即可。
</Callout>

## 单台设备的设置 modal

点卡片打开居中的「设备设置」modal，含以下控件：

### 基本信息

- 顶部 header 显示设备图标 + 名字 + 状态点（在线/离线）+ Peer ID 摘要。完整 Peer ID 在「连接信息」段。
- **连接信息** 段以圆角 InfoRow 显示 `通道`（直连 / 中转 / 未连接 / 不在本地网络）与（若有）`地址`。

### 发送总开关

<Feature since="0.7.0">
  按对端的 **出站门** 与 **内容类型 allowlist** 现在真正在调度器 侧执行（之前只是 UI
  持久化了但没生效）。被关掉的类型，对应字节 根本不会离开本机。
</Feature>

`向此设备发送（Send to this device）`：关掉后，本机产生的剪贴永远
不会发到这台对端；**接收始终保持开启**，对端仍可向本机推送剪贴。
独立的「接收」开关 **目前没开放给用户**，按计划在后续版本提供。这
是按对端的「出站」开关，不是空间级别。

### 内容类型白名单

发送总开关打开时，下方以 2 列 pill 网格列出每种内容类型的细分开关，每格 = 一个 icon + 标签 + 小 Switch：

- 文本
- 图片
- 链接
- 文件
- 代码（Code Snippet，标 「即将推出」）
- 富文本（HTML / RTF，标 「即将推出」）

激活的开关会用主色描边 + 浅色底色；发送总开关关掉时整个网格淡出且不可点。

任何被关掉的类型，对应字节 **永远不会离开本机**（在调度层就被挡掉）。这意味着：

- 关掉「文件」 → 不会再消耗对端的文件缓存配额。
- 关掉「图片」 → 大屏截图不会再吃带宽。
- `恢复默认` 是「同步设置」段右上角的 ghost 按钮（不在 footer），点一下把每个开关都回到默认值。

<Callout type="info">
  规则只对 **这一个对端**
  生效。一台设备配多对端时，可以分别设置，比如对工作笔记本只发文本，对手机不发文件等。
</Callout>

## 解除配对

modal footer 左侧的 `取消配对` 按钮（破坏性，会要求二次确认）。解除后：

- 这台对端从「空间成员」列表里消失。
- 后续对端发来的传输请求会被本机拒绝（基于 node ID）。
- **本机历史不动** —— 之前同步过来的内容仍在。

之后想让这台设备重新加入，直接重新走一遍 `invite` + `join`
即可 —— 解除配对是单侧操作，对端仍残留的旧记录会在新握手中被替换，
不会阻塞重新配对。

要从空间彻底移除一台 **丢失 / 失窃** 的设备：

1. 在另一台还在线、可信的设备上对它执行 `解除配对`。
2. 保险起见，在所有其它设备上各做一次（吊销目前是按设备本地名单维护的；分布式吊销在路线图上）。
3. 如果担心丢失设备本机仍有可解密历史，可考虑重建空间（详见 [常见问题 — 口令忘了能找回吗](../help/faq#口令忘了能找回吗)），但这会让所有设备重新配对一次。

## 移动端同步设备

已配对的移动客户端在「**手机同步**」tab 下，独占一片卡片网格 ——
它们不是桌面对端，也不属于 iroh 信任网络。

### Tab 头部的操作

切到「手机同步」tab 时，tab 行右侧出现两个按钮：

| 按钮       | 作用                                                                                                                            |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `配置`     | 打开 **移动端同步设置** modal —— 总开关、LAN 监听、监听 IP、端口、当前监听 URL 都在里面。                                       |
| `添加设备` | 走注册流程，详见 [移动端同步 — 配对一台 iPhone](./mobile-sync#配对一台-iphone)。未启用 / LAN 监听未开时点击会先弹出引导对话框。 |

监听器若 bind 失败（端口被占用、权限不足等），`添加设备` 按钮会变灰，
鼠标悬停可看到原因。完整原因 + LAN 安全告警在 **配置** modal 内部的
Alert 区域展示，不再有专门的状态条。

### 已配对的卡片

每张卡片：

- 顶部：蓝色 `Smartphone` 图标。
- 中部：注册时给的标签 + Basic Auth **用户名**（mono，可直接复制做排错）。
- 底部：最近一次活动时间。

**整张卡片可点**。点击会弹出 **移动端同步设备（Mobile sync device）**
对话框，而不是在卡片上就地触发某个动作。dialog 内部分两个视图：

- **Info 视图（默认）** —— 展示当前连接信息：base URL（旁边带网卡
  下拉，可重新挑要对外宣告的 IP），以及一张 QR ——「服务器只存
  Argon2 哈希，QR 没法再凭空拼出，点 Change password 签发新密码后
  再来扫」的占位；改密成功后才换成真正的 `uniclipboard://connect`
  二维码，并在下方多出一块 amber **凭据（Credentials）** 区域，
  自带一键 **备份** 按钮。底部按钮：左边 **Change password**，
  右边 **Revoke** + **Done**。
- **Rotate 视图** —— 点 **Change password** 进入。可以可选输入新
  密码（留空则自动生成），提交成功后 dialog 会自动切回 Info 视图，
  顺手把新 QR / amber 凭据呈现出来，方便立刻扫码 / 复制。旧的
  Argon2id 哈希被原子失效，旧凭据立即停用。

点 **Revoke** 会删除设备记录。携带旧凭据的后续请求会拿到
`401 Unauthorized`。

<Callout type="info">
  移动卡片 **不** 会出现 `LAN / Direct / Relay` 传输徽标 —— 移动端同步走的是 SyncClipboard
  HTTP（不是 iroh 传输），没有可比较的项。
</Callout>

完整的端到端配置流程（LAN 监听器、对外宣告 IP、iOS 快捷指令安装、安全注意
事项）单独整理在 [移动端同步](./mobile-sync)。配对动作本身在
[配对与同步 — 移动端伴侣](../guides/pairing#移动端伴侣) 也有简短说明。

## 在线状态多久刷新一次

设备页是 **推送驱动** 的。守护进程内部跑一个 keepalive worker 监测对端
上下线，并通过 `peers.changed` WebSocket 事件广播变化；页面只负责订阅
并重渲染。

- **对端重新上线**：~1 秒内反映出来。对端自己的 keepalive 拨回本机时
  事件立即触发。
- **对端断网**：~25–60 秒被检测到。来自守护进程自身的 keepalive tick
  与底层 QUIC idle 超时。

页面还会在以下两种情况主动拉一次（兜底，防止某次推送被漏掉）：

1. 本次会话第一次打开设备页时。
2. 切换标签后重新回到设备页时。

**不再有周期性轮询** —— 关掉设备页后这个页面的网络流量会归零。如果某个
状态明显不对（例如远端明明在跑但你看到 `Offline`），切换标签再回来强制
刷新一次；仍不对参见 [故障排查 — 对端不上线](../help/troubleshooting#对端不上线)。

## 与 CLI 的对应

| GUI 操作                | CLI 等价                                                           |
| ----------------------- | ------------------------------------------------------------------ |
| 「空间成员」列表        | `uniclip devices`（持久记录） / `uniclip members`（含在线状态）    |
| 添加设备 → 邀请码       | `uniclip invite`                                                   |
| 加入已有空间            | `uniclip join <code>`                                              |
| 复制 Peer ID            | `uniclip status --json` 中的 `network.endpoint`                    |
| 解除配对                | 当前 **仅 GUI** 提供专用命令；CLI 等价子命令在路线图上             |
| 按对端的内容类型白名单  | 当前 **仅 GUI** 提供；CLI 等价配置子命令在路线图上                 |
| 移动端同步 — 状态       | `uniclip mobile-sync status`                                       |
| 移动端同步 — 列出已配对 | `uniclip mobile-sync status`                                       |
| 移动端同步 — 注册设备   | `uniclip mobile-sync setup`（向导） 或 `mobile-sync add --label …` |
| 移动端同步 — 吊销设备   | `uniclip mobile-sync revoke <device-id>`                           |
