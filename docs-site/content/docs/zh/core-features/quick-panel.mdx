快捷面板是 UniClipboard 的键盘流主入口：一个独立的浮窗，列出当前空间的剪贴历史，按方向键定位、回车直接粘贴回原应用。整个流程不需要把焦点切到主窗口。

## 唤出与隐藏

| 平台            | 默认快捷键       |
| --------------- | ---------------- |
| macOS           | `Cmd + Ctrl + V` |
| Windows / Linux | `Ctrl + Alt + V` |

按一次唤出，再按一次隐藏。`Esc` 也可以直接收起面板。面板失焦后会自动隐藏（设计上不抢长焦点）。

<Callout type="info">
  快捷键被其它应用占用了？面板"按了没反应"通常是热键冲突，不是 bug。到 [设置 —
  快捷键](../guides/settings#快捷键)改成空闲组合即可。如果根本不想要这个面板，也可以在 [设置 —
  快捷面板](../guides/settings#快捷面板) 里整体关掉它 —— 关闭即时反注册全局快捷键。
</Callout>

## 键盘操作

面板设计为完全用键盘驱动。下面这套键位可在打开后立即用：

| 按键                     | 作用                                    |
| ------------------------ | --------------------------------------- |
| `↑` / `↓`                | 上下移动选中项                          |
| `Enter`                  | 把选中项粘贴回上一个应用                |
| 按住 `Option` / `Alt`    | 请求纯文本粘贴                          |
| `Esc`                    | 关闭面板，不粘贴                        |
| `Alt + Backspace`        | 从历史中删除选中项                      |
| `Cmd` / `Ctrl` + `1`–`9` | 直接选第 1–9 项                         |
| `Cmd` / `Ctrl` + `0`     | 直接选第 10 项                          |
| 直接输入                 | 即时按子串过滤（substring，大小写无关） |
| `:`（行首）              | 切到高级搜索模式（见下文）              |

`Enter` 的粘贴是通过模拟系统粘贴完成的，所以接收方应用感知到的是"用户刚按了 Cmd/Ctrl+V"，富文本与文件类型也会按目标应用支持的最高保真表示落地。

如果想粘贴纯文本，确认条目时按住 `Option` / `Alt`：可以按 `Enter`、点击条目，或继续使用
`Cmd` / `Ctrl` + 数字快捷键。UniClipboard 会先只把 `text/plain` 表示写回系统剪贴板；如果条目没有纯文本表示，则静默回退到普通多格式粘贴。

## 内嵌预览

选中一项后，右侧预览面板会按内容类型自动选择渲染方式：

| 表示类型    | 预览形态                                |
| ----------- | --------------------------------------- |
| 纯文本      | 等宽字体；超大文本（>50KB）走虚拟化滚动 |
| 链接        | 提取域名 + 完整 URL                     |
| 图片        | 内嵌缩略，按比例适配预览区，附带尺寸    |
| 代码 / HTML | 与文本一致的渲染（不做语法高亮）        |
| 文件        | 文件名列表 + 图标，多文件按行排列       |

预览 **只读**。要把项目放回系统剪贴板或粘贴出去，回到列表按 `Enter`。

## 类型与时间筛选

面板顶部有两组筛选控件，互不冲突，可叠加：

- **类型筹码**：全部 / 文本 / 图片 / 链接 / 文件 / 代码。
- **时间下拉**：今天、昨天、近 7 天、近 30 天、本周、本月，或不限。

筛选写在内存里，关闭面板后 **不持久**——下次打开是默认全部。这是刻意的：面板是即用即抛的入口，不当工作台用。

## 高级搜索（`:` 模式）

在搜索框行首输入 `:`，面板进入高级模式，支持把过滤条件写进查询里：

```text
:type:image 2026
:type:code ext:ts useEffect
:ext:pdf 报告
```

支持的标记：

- `type:<text|image|link|file|code>` —— 限定内容类型，等价于点筹码。
- `ext:<扩展名>` —— 限定文件扩展名（仅对文件类负载生效）。
- 其余 token 作为子串匹配条件，多 token 之间是 AND。

不支持引号短语、布尔操作符、正则 —— 详见 [全文搜索 — 查询语法](./search#查询语法)。

<Callout type="info">
  CLI 端的等价能力是 `uniclip search query`，参数与上面对应：`--type` / `--ext` / `--from-ms` /
  `--to-ms`。详见 [CLI 参考 — 搜索](../cli/reference#搜索)。
</Callout>

## 何时用面板，何时回主窗口

| 你想做的事                         | 推荐入口                                       |
| ---------------------------------- | ---------------------------------------------- |
| 立刻粘贴最近的某一项               | 快捷面板                                       |
| 跨多设备查找几天前的内容           | 快捷面板的 `:` 高级搜索 / CLI `uniclip search` |
| 长时间整理、批量删除、导出         | 主窗口仪表盘                                   |
| 配置筛选规则、按设备的内容类型限制 | [设备页](./devices) / [设置](../guides/settings)       |

## 排查

- 按了快捷键没反应：先看 [故障排查 — 剪贴板权限](../help/troubleshooting#剪贴板权限)，macOS 还要确认 **输入监控** 已授权并彻底重启过应用。
- 面板里看不到刚复制的内容：守护进程没起或剪贴板监听被系统隔离。`uniclip status` 看 `daemon.running` 与剪贴板监听日志。
- 同步过来的项目从未出现在面板：可能在发送端被设备规则挡掉了，到对端的 [设备 → 内容类型白名单](./devices#内容类型白名单) 检查。
