UniClipboard 给本地剪贴历史维护了一份 **加密的全文搜索索引**，"存在本地"和"存得安全"两件事是同时成立的。索引在磁盘上加密，密钥与历史数据库共用一套，详见 [同步内容 — 本地存储](./sync#本地存储)。

## 在哪里搜

| 入口                 | 当前状态 / 适用场景                                                                                                               |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| 历史页面             | **应用内主入口。** 历史视图现在是落地页，其搜索框把自由文本与「内容类型 / 来源设备 / 时间」筛选揉在一起，接入下文同一套查询服务。 |
| 快捷面板             | 全局热键、键盘流的快速入口。即时筛选、高级语法。详见 [快捷面板](./quick-panel#高级搜索-模式)                                      |
| `uniclip search` CLI | 脚本、SSH、远程会话。详见 [CLI — 搜索](../cli/reference#搜索)                                                                     |

## 查询语法

快捷面板的高级搜索、历史页面的搜索框、CLI 三者背后用的是同一个查询解析器，所以下面的 token 规则三处通用 —— 唯一例外是「来源设备」筛选，只有历史页面的搜索框支持（见下文）。

### 子串匹配

不带任何前缀的 token 都会被当作大小写无关的子串过滤条件。多个 token 之间是 AND：

```text
报告 v2
project meeting notes
```

排序基于 BM25 相关度，不是按时间。需要时间序请改用时间筛选（见下文）。

### 类型过滤 `type:`

```text
type:text 邀请
type:image 截图
type:code useEffect
type:link mdn
type:file invoice
```

等价于快捷面板顶部的类型筹码、CLI 的 `--type`。可选值：`text` / `image` / `link` / `file` / `code`。HTML 现已是索引层独立的内容类别（原先合并的 `code` 类已不存在）。

### 扩展名过滤 `ext:`

```text
ext:pdf 合同
ext:ts useReducer
```

只对 **文件类** 负载生效（基于文件名后缀）。CLI 等价是 `--ext`。

### 时间范围

面板里通过下拉选预设（今天 / 近 7 天 / 本月 等），CLI 中用 `--from-ms` / `--to-ms` 指定毫秒时间戳：

```bash
uniclip search "报告" \
    --from-ms 1710000000000 \
    --to-ms 1710100000000
```

### 来源设备过滤

把结果收窄到来自某一台设备的内容 —— 可以是配对的电脑，也可以是 mobile-sync 的手机。在历史页面的搜索框里，从筛选栏选一台设备，或输入 `from:` 再从列表里挑。这个筛选只在历史视图里有；快捷面板和 `uniclip search` 没有对应能力。

### 标签过滤 `#`

把结果收窄到带某个内容标签的内容。标签在条目入索引时自动派生：

- `#link` —— 内容里含网页 URL。
- `#code` —— 内容是富文本/HTML，或看起来像源代码的纯文本。

在历史页面的搜索框里，输入 `#` 再从列表里挑标签。和「来源设备」筛选一样，这个能力只在历史视图里有；快捷面板和 `uniclip search` 没有对应能力。注意 `#code`（内容标签）与上文的 `type:code`（内容类型）是两个不同维度。

### **不** 支持的语法

下列在很多搜索框里习以为常的特性，UniClipboard 当前不支持：

- 引号短语（`"exact phrase"`）—— 引号被当作普通字符处理。
- 布尔操作符（`AND` / `OR` / `NOT`）—— `OR` 会被当成子串。
- 通配符 / 正则 —— 同上。
- 在内容类型之间做布尔组合（如 "text 或 link"）。

需要更复杂的查询时，建议把 `--detailed --json` 输出导出，再在外部脚本里过滤。

## 索引生命周期

索引随守护进程启动而打开，复制的内容在进入加密历史的同时就会被分词并加索引，对查询结果是 **接近实时** 的可见。但有几个时刻你会显式碰到它：

### 首次启动 / 大版本升级后

索引可能处在"重建中"状态。

- **GUI**：**设置 → 存储 → 搜索索引** 区域显示当前状态（就绪 / 重建中 / 不可用）、上次重建时间、`重建`按钮。
- **CLI**：`uniclip search status` 给出同样的状态字段。

### 手动重建

少数情况下索引会和历史漂出一致（最常见：进程在写索引时被 SIGKILL）。

被中断的重建现在会在守护进程下次启动时（例如重新打开应用）自动续上，所以多数情况下无需手动点重建 —— 除非你不想等一次重启。

- **GUI**：上面那个 `重建` 按钮，点了之后状态变"重建中"，每 2 秒轮询一次直到结束。
- **CLI**：`uniclip search rebuild` 是同步命令 —— 直到重建完成才返回，便于脚本里抓诊断快照。

重建过程中查询不会失败，只是返回的结果集合在补齐之前是不完整的。

### 损坏修复

索引文件结构受损（极罕见，比如磁盘错误）时，单纯重建可能起不来。删除数据目录下的索引子目录、再 `rebuild` 即可重新从历史构建一份。剪贴历史本身存在另一个文件，**不会受影响**。

详见 [故障排查 — 搜索与索引](../help/troubleshooting#搜索与索引)。

## 隐私

- 索引文件在磁盘上加密，密钥放在系统密钥环，与历史数据库共用 KEK / MasterKey。
- 索引 **不离开本机**，也不参与跨设备同步。每台设备搜索的是本机看到过的内容。
- 这意味着新加入的设备搜不到加入前的历史 —— 这是 [快速开始 — 加入不回填历史](../getting-started/quick-start#添加第二台设备) 那一行设计的直接后果。

## 查不到内容怎么办

按可能性从大到小：

1. 拼写或大小写问题。直接换个更短的 token 试试 —— 子串匹配对噪音敏感。
2. 时间过滤太窄。把时间下拉切回 **不限** 再说。
3. 该条目在历史保留策略里被 GC 了。看 [设置 — 存储](../guides/settings#存储) 里的最大条目数 / 保留天数，确认还在窗口里。
4. 该条目压根没同步过来 —— 看 [设备 — 内容类型白名单](./devices#内容类型白名单) 是否在发送端被挡掉。
5. 索引尚未就绪。看 GUI 的状态指示灯，或跑 `uniclip search status`。
