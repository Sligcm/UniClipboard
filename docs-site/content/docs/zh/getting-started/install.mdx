UniClipboard 为 Windows、macOS、Linux 提供桌面应用与 `uniclip` 命令行工具。按你的平台选择安装方式 ——
所有安装包最终指向同一组
[GitHub Releases](https://github.com/UniClipboard/UniClipboard/releases)
构建产物。带 `-alpha.N` 后缀的 tag 是预发布版本，会在 Releases 页面被
标记为「Pre-release」；不带后缀的 tag 即正式发布。

## 选择平台

<Tabs items={['macOS', 'Windows', 'Linux']}>

<Tab value="macOS">

macOS 推荐使用官方 Homebrew tap。

```bash
brew tap UniClipboard/tap

# 桌面应用（.app）
brew install --cask uniclipboard

# 仅安装 CLI，命令名为 `uniclip`
brew install uniclipboard
```

也可以省去 `brew tap`，一行直装：

```bash
brew install --cask UniClipboard/tap/uniclipboard   # GUI
brew install UniClipboard/tap/uniclipboard          # CLI
```

cask 与 formula 互不冲突，需要的话两个都装即可（GUI + 终端 `uniclip`）。

更习惯手动安装？前往
[Releases](https://github.com/UniClipboard/UniClipboard/releases)
页面下载最新 `.dmg`，把应用拖入 `/Applications`。

### 首次启动权限

首次启动时 macOS 会请求两项权限，两者都是剪贴板同步必需的：

- **辅助功能（Accessibility）** —— 用于读取系统剪贴板。
- **输入监控（Input Monitoring）** —— 用于全局快捷面板快捷键。

如果稍后才在 **系统设置 → 隐私与安全** 中授予，记得退出应用再重新启动以使权限生效。

</Tab>

<Tab value="Windows">

Windows 提供两种形态：

- **MSI 安装包** —— `UniClipboard_<version>_x64_en-US.msi`。标准安装流程，会写入开始菜单与卸载项。
- **便携版 ZIP** —— 解压到任意位置，运行 `UniClipboard.exe`。不写注册表，适合 U 盘或受限环境。

两种产物都已数字签名。

按需从
[Releases](https://github.com/UniClipboard/UniClipboard/releases)
页面下载。

<Callout type="info">
  首次运行新版本时 SmartScreen 可能弹出警告。如果发布者信息与签名身份一致，点击 **更多信息 →
  仍要运行**。如果不放心，可以在 **属性 → 数字签名** 中校验签名。
</Callout>

</Tab>

<Tab value="Linux">

每次发布都会构建 `.deb`、`.rpm` 与 `.AppImage` 三种格式，覆盖 `x86_64` 与 `aarch64`。按你的发行版挑最合适的渠道：

### Fedora / RHEL / openSUSE — 推荐 COPR

通过 [Fedora COPR](https://copr.fedorainfracloud.org/coprs/mkdir700/) 一键安装并自动跟随版本升级：

```bash
# Alpha 渠道（含预发布版本，更新更频繁）
sudo dnf copr enable mkdir700/uniclipboard-alpha
sudo dnf install uniclipboard

# 正式渠道（仅稳定版）
sudo dnf copr enable mkdir700/uniclipboard
sudo dnf install uniclipboard
```

启用后 `sudo dnf upgrade` 会自动拉取新版本。详见 [COPR 部署手册](https://github.com/UniClipboard/UniClipboard/blob/main/docs/packaging/COPR.md)。

### Ubuntu / Debian / 老发行版 — 推荐 Snap

Snap 包基于 `core22` 严格沙箱构建，自带 GTK + WebKit 运行时，因此在 Ubuntu 20.04+、Debian 11+、RHEL 9、openSUSE Leap 等老发行版上都能正常运行。

```bash
sudo snap install uniclipboard
```

没有 `snapd`？请参考 [Snapcraft 安装指南](https://snapcraft.io/docs/installing-snapd) 查看你所用发行版的安装方式。

### 任意发行版 — AppImage

每个 release 也会发布 AppImage。从 [Releases](https://github.com/UniClipboard/UniClipboard/releases) 页面下载，赋予可执行权限即可运行：

```bash
chmod +x UniClipboard_<version>_amd64.AppImage
./UniClipboard_<version>_amd64.AppImage
```

AppImage 是 Linux 上 App 内更新器唯一支持的自更新格式 —— 走 COPR / Snap / dpkg 安装的版本不会通过应用内更新器升级，请用对应包管理器。

### 一次性手动安装单个 .deb / .rpm

不想加仓库？也可以从 Releases 直接下单文件安装：

```bash
# Debian / Ubuntu
sudo dpkg -i uniclipboard_<version>_amd64.deb
sudo apt-get install -f                                 # 如有缺失依赖，由 apt 补齐

# Fedora / RHEL / openSUSE（一次性，不加 COPR 仓库）
sudo dnf install ./UniClipboard-<version>-1.x86_64.rpm
```

### 无桌面环境（仅安装 CLI）

服务器、SSH 会话或任何没有桌面的环境，可以从 Releases 单独下载 `uniclip` CLI —— 查找 `uniclip-<version>-<arch>` 产物（与 musl 静态链接）。CLI 内置守护进程，无需 GUI 即可运行。

</Tab>

## 从源码构建

需要 [Bun](https://bun.sh) 与 Rust 工具链（`rustup default stable`）。

<Steps>

<Step>

### 克隆仓库

```bash
git clone https://github.com/UniClipboard/UniClipboard.git
cd UniClipboard
```

</Step>

<Step>

### 安装依赖

```bash
bun install
```

</Step>

<Step>

### 开发模式启动，或构建发行包

```bash
# 本地开发，桌面应用支持热重载
bun tauri dev

# 为当前平台构建发行包
bun tauri build
```

</Step>

</Steps>

发行包位于 `src-tauri/target/release/bundle/`。分支策略、commit 规范与 PR 流程见
[CONTRIBUTING_ZH.md](https://github.com/UniClipboard/UniClipboard/blob/main/CONTRIBUTING_ZH.md)。

## 验证安装

安装完成后，确认 CLI 已在 `PATH` 中：

```bash
uniclip --version
```

对于 GUI，首次启动应显示 **新建空间** / **加入已有空间** 两个选项；如果已有空间，则直接进入仪表盘。

## 数据存储位置

UniClipboard 把加密数据库、搜索索引与日志放在各平台标准的应用数据目录下，目录名为
`app.uniclipboard.desktop`：

| 平台    | 路径                                                      |
| ------- | --------------------------------------------------------- |
| macOS   | `~/Library/Application Support/app.uniclipboard.desktop/` |
| Linux   | `~/.local/share/app.uniclipboard.desktop/`                |
| Windows | `%LOCALAPPDATA%\app.uniclipboard.desktop\`                |

日志位于其下的 `logs/` 子目录。设置 `UC_PROFILE` 环境变量（如
`UC_PROFILE=dev`）可在目录名后追加后缀（变成
`app.uniclipboard.desktop-dev`），这样开发版与正式版可以并存。
