UniClipboard 通过 **空间（Space）** 进行同步 —— 一个由你的设备共同组成的、端到端加密的私有群组。第一台设备创建空间，之后每台设备都通过短期邀请码加入这个空间。没有云账号、没有邮箱 —— **空间本身就是账号**。

本页带你从两台机器全新安装走到第一次成功跨设备粘贴。

## 准备工作

- 至少在 **两台设备** 上完成安装。下载方式见[安装](./install)。
- 一个能记住的口令。它保护空间内所有剪贴内容，并且 **无法找回** —— 这是密码学设计的一部分，不是 bug。
- 两台设备同时在线。任意网络环境都可以：同一 Wi-Fi、不同家庭网络、甚至手机热点。能直连就直连，连不通时自动回落到加密中继。

<Callout type="info">
  信任边界是 **空间**，而不是单台设备。后续添加第三台、第四台设备的流程与添加第二台完全一致 ——
  生成邀请码、在新设备上输入。
</Callout>

## 设置第一台设备

<Steps>

<Step>

### 启动应用，选择 **新建空间**

首次启动会看到两个选项：**新建空间** 与 **加入已有空间**。选第一个。

</Step>

<Step>

### 设置空间口令

口令通过 Argon2id 派生加密密钥，再由 XChaCha20-Poly1305 加密每一条剪贴内容。请使用足够长且容易记的口令 —— 这里没有"忘记密码"流程，是刻意设计：你的空间内容从不以明文形式离开过设备。

</Step>

<Step>

### 开始复制

仪表盘打开后，你复制的任何内容都会被捕获进空间，等待下一台设备加入即可同步。

</Step>

</Steps>

## 添加第二台设备

把两台设备都准备在手边 —— 一台生成邀请码，一台输入邀请码。

<Tabs items={['桌面端', '命令行（uniclip）']}>

<Tab value="桌面端">

<Steps>

<Step>

### 在 **第一台** 设备 → **设备** → **生成邀请码**

邀请码短期有效（几分钟内可用），仅一次性。它 **不** 包含你的口令。

</Step>

<Step>

### 在 **新** 设备 → **加入已有空间**

输入邀请码，并填入你在第一台设备上设置的空间口令。口令在本地验证 —— 输错会立即失败，且永远不会发送给另一台设备。

</Step>

<Step>

### 等待首次同步

加入完成后，新设备会自动开始同步历史。两台设备的 **设备** 页都会出现对方，并显示在线状态。

</Step>

</Steps>

</Tab>

<Tab value="命令行（uniclip）">

CLI 与 GUI 流程完全一致，适用于无桌面环境（如服务器、SSH 会话、tmux 工作流）。

```bash
# 在第一台设备上（已在 GUI 创建过空间则跳过 init）
uniclip init

# 生成短期邀请码
uniclip invite

# 在新设备上输入上一步生成的邀请码
uniclip join <code>

# 确认双方互相看见
uniclip members
```

守护进程会自动启动。可用 `uniclip status` 查看状态，或 `uniclip start` / `uniclip stop` 显式控制。

</Tab>

</Tabs>

## 验证同步

1. 在设备 A 复制任意内容 —— 文本、图片或文件均可。
2. 切到设备 B 用 `Cmd+V` / `Ctrl+V` 粘贴，或用快捷键唤出 **快捷面板** 直接挑选记录。

如果粘贴立刻出现，说明同步已经跑通。如果没有，[配对与同步](../guides/pairing)介绍了传输细节（P2P 打洞、加密中继回落）以及如何排查"卡住"的对端。

## 添加手机（可选）

UniClipboard 还提供一种 **实验性的 HTTP 伴侣** 模式，用于在桌面与手机
之间互相粘贴。移动端走的是和桌面 **不同** 的配对路径 —— 它不加入空间的
iroh 信任网络、不分配 node ID。默认在 LAN 下与桌面互通，需要跨网络时
可经自建 server 节点或 Tailscale。

1. 在桌面端打开 **设备 → 添加 → 移动设备同步**，给这台设备起一个标签。
2. 凭据弹窗里按 **接入方式** 拆成两个 tab：
   - **扫码接入（Scan to add）** —— 一个 connect-URI 二维码。在
     UniClipboard iOS App（或任意兼容 SyncClipboard 协议的客户端，
     含社区维护的安卓 App）里扫一下，base URL、用户名、密码自动填好。
   - **安装快捷指令（Install Shortcut）** —— 在用户没装 UniClipboard
     iOS App 时走这条：扫码安装 `UniClipboard.shortcut`，再 **手动** 把
     弹窗里的 base URL、用户名、密码粘贴到快捷指令的对应字段。
   - 安卓侧装 **UniClipboard Android 客户端**：
     [github.com/UniClipboard/uc-android](https://github.com/UniClipboard/uc-android)
     —— fork 自 SyncClipboard，APK 在
     [releases 页](https://github.com/UniClipboard/uc-android/releases/latest)
     下载（侧载，不发布到 Google Play）。
3. 明文密码 **只显示一次**。若遗失，可以在设备行使用 **轮换密码** 重新签发，
   无需重新配对。

完整流程与协议细节见
[移动端同步](../core-features/mobile-sync)（端到端配置）、
[配对与同步 — 移动端伴侣](../guides/pairing#移动端伴侣) 与
[移动端 LAN API](../reference/mobile-api)。

## 接下来

- **添加更多设备。** 在每台你想加入空间的机器上重复"添加第二台设备"。每个邀请码都是一次性的。
- **试试快捷面板。** 用配置的快捷键唤出，键盘流即时取用最近的剪贴 —— 文本、链接、图片、代码与文件。
- **切换空间。** 在 **设备** 页或带上另一个 sponsor 的邀请码运行 `uniclip join --switch` 切换，例如把个人机和工作机分到不同的空间。切换时本地历史会用新空间的密钥重新加密。
- **接入手机（实验性）。** 端到端流程见
  [移动端同步](../core-features/mobile-sync)。

  从 0.6
  之前的版本升级而来？底层网络栈做了重写，旧的配对状态已不再有效。重新生成一次邀请码、走一次加入流程即可恢复同步。
</Callout>
