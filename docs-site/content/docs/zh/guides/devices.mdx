设备页是空间成员的实时视图：哪台机器在线、走的是直连还是中继、哪些内容类型被允许从这边发过去。所有动作都对 **当前空间** 生效；切换空间后视图会随空间更新。

## 页面结构

| 区块               | 内容                                                                                                               |
| ------------------ | ------------------------------------------------------------------------------------------------------------------ |
| 「这台设备」卡片   | 本机名称、当前同步状态（活动 / 暂停）、已配对设备数、其中在线数                                                    |
| 「空间成员」面板   | 当前空间内所有已配对的桌面对端设备列表，每行可展开查看详细                                                         |
| 「移动端同步」面板 | LAN 监听器状态、`Configure` 配置抽屉、`Add device` 入口，以及已配对的移动客户端列表。详见[下文](#移动端同步设备)。 |

「这台设备」是只读摘要，要改本机名 / 同步开关请去 [设置 → 通用](./settings#通用) 与 [设置 → 同步](./settings#同步)。

完整的移动端同步配置流程（LAN 监听器、iOS 快捷指令、凭据管理）单独整理在
[移动端同步](./mobile-sync) 这一节。

## 成员行能告诉你什么

每行从左到右：

- **设备图标** —— 根据设备名启发式选色（在线为强调色、离线为灰）。
- **设备名** —— 该设备本机起的名字；过长会被截断。
- **状态徽标** —— `Online`（含传输通道，见下文）或 `Offline`。
- **箭头** —— 点开右侧设备设置抽屉。

### 传输通道徽标

<Feature since="0.7.0">
  按对端展示 `LAN` / `Direct` / `Relay` 的可达性徽标。底层由 `ConnectionChannelPort`
  单一真相源贯穿到 DTO，GUI、托盘图标与 `uniclip status --json` 三处看到的状态一致。
</Feature>

`Online` 旁的小标签告诉你这次连接走的是哪条路：

| 标签     | 含义                              |
| -------- | --------------------------------- |
| `LAN`    | 通过局域网 mDNS 发现并直连。      |
| `Direct` | NAT 穿透成功，跨网络的 P2P 直连。 |
| `Relay`  | P2P 打洞失败，走加密中继转发。    |

中继看到的永远是密文，参见 [配对与同步 — 中继看到的](./pairing#中继看到的-vs-对端看到的)。**这个标签只反映可达性 / 性能层面**，不影响机密性。

## 添加新设备

在「空间成员」面板右上角点 `添加设备`。弹窗会：

1. 调用 sponsor 守护进程生成一段一次性邀请码。
2. 显示邀请码 + 倒计时（默认有效期 5 分钟）。
3. 提供一键复制按钮。

把邀请码送到新设备（同物理位置直接念 / 私聊都行 —— 邀请码不带口令），让对方在新设备上点 **加入已有空间** → 输入邀请码与空间口令。详见 [快速开始 — 添加第二台设备](../getting-started/quick-start#添加第二台设备)。

<Callout type="warn">
  邀请码过期或已被消费就 **无法再用**。这是配对协议的"一次性"性质决定的，不是 bug ——
  重新生成一张即可。
</Callout>

## 单台设备的设置抽屉

点成员行右侧箭头会从右侧滑出抽屉，含以下控件：

### 基本信息

- **Peer ID** —— 该设备的 iroh node ID（公钥），可点 `复制` 用作 issue 报告或诊断材料。
- **设备名** —— 只读，由对端本机控制。

### 发送总开关

<Feature since="0.7.0">
  按对端的 **出站门** 与 **内容类型 allowlist** 现在真正在调度器 侧执行（之前只是 UI
  持久化了但没生效）。被关掉的类型，对应字节 根本不会离开本机。
</Feature>

`向此设备发送（Send to this device）`：关掉后，本机产生的剪贴永远
不会发到这台对端；**接收始终保持开启**，对端仍可向本机推送剪贴。
独立的「接收」开关 **目前没开放给用户**，按计划在后续版本提供。这
是按对端的「出站」开关，不是空间级别。

### 内容类型白名单

发送总开关打开时，下面会列出每种内容类型的细分开关：

- 文本
- 图片
- 链接
- 文件
- 代码（Code Snippet）
- 富文本（HTML / RTF）

任何被关掉的类型，对应字节 **永远不会离开本机**（在调度层就被挡掉）。这意味着：

- 关掉「文件」 → 不会再消耗对端的文件缓存配额。
- 关掉「图片」 → 大屏截图不会再吃带宽。
- 改回默认用抽屉底部的 `恢复默认`，等价于全打开。

<Callout type="info">
  规则只对 **这一个对端**
  生效。一台设备配多对端时，可以分别设置，比如对工作笔记本只发文本，对手机不发文件等。
</Callout>

## 解除配对

抽屉底部的 `解除配对` 按钮（破坏性，会要求二次确认）。解除后：

- 这台对端从「空间成员」列表里消失。
- 后续对端发来的传输请求会被本机拒绝（基于 node ID）。
- **本机历史不动** —— 之前同步过来的内容仍在。

要从空间彻底移除一台 **丢失 / 失窃** 的设备：

1. 在另一台还在线、可信的设备上对它执行 `解除配对`。
2. 保险起见，在所有其它设备上各做一次（吊销目前是按设备本地名单维护的；分布式吊销在路线图上）。
3. 如果担心丢失设备本机仍有可解密历史，可考虑重建空间（详见 [常见问题 — 口令忘了能找回吗](../faq#口令忘了能找回吗)），但这会让所有设备重新配对一次。

## 移动端同步设备

已配对的移动客户端会显示在同一页面的 **「移动端同步」独立面板** 中，与
「空间成员」列表分开 —— 它们不是桌面对端，也不属于 iroh 信任网络。

### 状态条

面板顶端的状态条一眼反映监听器当前状态：

| 状态                      | 含义                                                                             |
| ------------------------- | -------------------------------------------------------------------------------- |
| `Disabled`（已停用）      | 总开关未开。点 `Configure` 进配置抽屉打开它。                                    |
| `Listening @ …`（监听中） | LAN 监听器在运行，下面显示的 URL 就是手机端要填的目标。                          |
| `Bind failed: …`          | daemon 试图 bind 但失败（如端口占用、权限不足），原因来自 `lan_listener_error`。 |
| `Restart required`        | 配置变了（监听 IP、端口、总开关），需要重启 daemon 才会生效。                    |

`Configure` 打开 **移动端同步设置** 抽屉（详见
[设置 → 移动端同步](./settings#移动端同步)）。`Add device` 走注册流程，详见
[移动端同步 — 配对一台设备](./mobile-sync#配对一台设备)。

### 已配对设备列表

每行包含：

- 注册时给这台设备起的标签。
- Basic Auth 的 **用户名**（作为稳定标识；排错时可直接复制）。
- 最近一次在线时间、最近一次 IP，以及客户端上报的设备信息（例如
  `iOS` / `iOS Shortcut`）。
- **轮换密码** 按钮 —— 签发一个新的一次性密码，并原子失效旧的 Argon2id
  哈希。旧凭据立即停用；新密码只显示一次。
- **吊销** 按钮 —— 删除设备记录。携带旧凭据的后续请求会拿到
  `401 Unauthorized`。

<Callout type="info">
  移动端行 **不** 会出现上面提到的 `LAN / Direct / Relay` 传输徽标 —— 移动端同步 **总是**
  走局域网内的明文 HTTP，没有可比较的项。
</Callout>

完整的端到端配置流程（LAN 监听器、对外宣告 IP、iOS 快捷指令安装、安全注意
事项）单独整理在 [移动端同步](./mobile-sync)。配对动作本身在
[配对与同步 — 移动端伴侣](./pairing#移动端伴侣仅限-lan) 也有简短说明。

## 在线状态多久刷新一次

设备页的状态来源有两个：

- **轮询**：每 15 秒主动拉一次成员列表。
- **事件驱动**：守护进程的 `peers.changed` WebSocket 事件触发即时刷新。

正常情况下变化在事件触发的瞬间就反映；轮询是兜底。如果状态明显不对（例如远端明明在跑但你看到 `Offline`），等 15 秒再判断一次，仍不对参见 [故障排查 — 对端不上线](./troubleshooting#对端不上线)。

## 与 CLI 的对应

| GUI 操作                | CLI 等价                                                                   |
| ----------------------- | -------------------------------------------------------------------------- |
| 「空间成员」列表        | `uniclip devices`（持久记录） / `uniclip members`（含在线状态）            |
| 添加设备 → 邀请码       | `uniclip invite`                                                           |
| 加入已有空间            | `uniclip join <code>`                                                      |
| 复制 Peer ID            | `uniclip status --json` 中的 `network.endpoint`                            |
| 解除配对                | 当前 **仅 GUI** 提供专用命令；CLI 等价子命令在路线图上                     |
| 按对端的内容类型白名单  | 当前 **仅 GUI** 提供；CLI 等价配置子命令在路线图上                         |
| 移动端同步 — 状态       | `uniclip mobile-sync status`                                               |
| 移动端同步 — 列出已配对 | `uniclip mobile-sync devices list`                                         |
| 移动端同步 — 注册设备   | `uniclip mobile-sync setup`（向导） 或 `mobile-sync devices add --label …` |
| 移动端同步 — 吊销设备   | `uniclip mobile-sync devices revoke <device-id>`                           |
