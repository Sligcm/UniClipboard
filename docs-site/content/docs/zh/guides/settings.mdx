设置面板按主题分为八组（外加一个「关于」）。下面按面板里的左右顺序逐组列出，每项给出 **作用** 和 **何时该改它**。

移动端同步的配置 **不在** 设置面板里，而是放在 **设备** 页的独立弹窗中
—— 单独整理在 [下文](#移动端同步) 与 [移动端同步](../core-features/mobile-sync) 一节。

## 通用

| 设置               | 作用                                                                                                                                         |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 开机自启           | 系统登录时自动启动 UniClipboard。                                                                                                            |
| 静默启动           | 启动后不弹窗，直接进入后台 / 托盘。                                                                                                          |
| 设备名             | 显示在其它设备的「成员列表」里 —— 起一个人能认的名字。                                                                                       |
| 语言               | 当前支持英文 / 简体中文。                                                                                                                    |
| 隐私：诊断数据     | 是否发送崩溃、错误、匿名日志与前端诊断数据；默认开启，可随时关闭。                                                                           |
| 隐私：使用情况分析 | 是否发送匿名产品事件（激活漏斗、配对 / 同步可靠性）；默认开启，可单独关闭。                                                                  |
| 调试模式           | 提高本地诊断日志的详细程度。开启时会弹出确认框，确认后会自动重启应用与 daemon 使其生效；关闭后需手动重启才能完全恢复普通日志级别。默认关闭。 |
| 导出日志           | 把最近 24 小时的 GUI、daemon、CLI 日志打包成 zip 放到下载目录，便于提交 issue。                                                              |

设备名只是显示用的别名，不参与加密身份。改名不会影响 node ID 或配对状态。

诊断数据与使用情况分析是两个独立开关。它们不包含剪贴板内容、文件内容、口令或密钥；完整字段范围见 [隐私与数据收集](../core-features/privacy)。

## 外观

| 设置     | 作用                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 主题模式 | 浅色 / 深色 / 跟随系统。切换即时生效。                                       |
| 主题色   | 调色盘选择强调色（slate, rose, orange, amber, ..., blue, violet 等）。       |
| 界面缩放 | 80% / 90% / 100% / 110% / 120% 几档可选，也可拖滑块自由调；`重置`回到 100%。 |

缩放只影响渲染密度（DPI 概念），不影响数据；多设备之间不会同步。

## 快捷键

按"作用域"分组列出可绑定的命令，每一项都可以单独覆盖、重置或清空：

- **全局**：跨进程生效，包括 [快捷面板](../core-features/quick-panel) 的唤起键。
- **剪贴**：主仪表盘里浏览历史时的导航键。
- **设置**：设置面板内部导航。
- **设备**：设备页内的导航。
- **模态**：通用对话框 / 抽屉操作。

录制框直接按下你想用的组合即可，冲突会即时高亮提示。`重置`回默认，`清空`让该项无绑定。

## 快捷面板

| 设置         | 作用                                                                                                                      |
| ------------ | ------------------------------------------------------------------------------------------------------------------------- |
| 启用快捷面板 | 总开关。开启即时注册全局快捷键并在后台预创建面板窗口；关闭即时反注册快捷键，立刻不再响应。                                |
| 展示位置     | 触发时面板出现的位置 —— **屏幕中央**（默认）或 **跟随光标**（靠近指针落点，自动避开屏幕边缘）。仅影响下次展示，无需重启。 |

默认开启 —— [快捷面板](../core-features/quick-panel) 是产品的键盘流主入口。
如果默认快捷键与其它你不能改的应用冲突，或者你所在的环境
不允许常驻后台面板窗口，再考虑关掉它。

重新开启完全即时：切到 on 立刻注册快捷键并预创建窗口。**关闭** 则只有
一半即时 —— 全局快捷键立刻失效，但已经创建好的后台窗口（及其 WebContent
进程）会一直驻留到你重启应用为止。开关上方会出现一行 banner，提供一键重启入口，
方便彻底回收资源。

快捷键本身在 [快捷键 → 全局 → 唤起快捷面板](#快捷键)，本节中再次镜像同一行，
让你在同一处完成"开关 + 配快捷键"两件事；两处共用同一份绑定。

## 同步

控制本机参与同步的方式与文件相关阈值：

| 设置                 | 作用                                                   |
| -------------------- | ------------------------------------------------------ |
| 启用自动同步         | 关闭后这台设备只接收，不主动推送（仍可手动 `send`）。  |
| 启用文件同步         | 总开关；关掉时文件类负载不再传出 / 传入。              |
| 小文件阈值（MB）     | 低于此阈值的文件按内联编码走快通道；高于的进 blob 流。 |
| 单文件最大值（MB）   | 超过此值会被拒绝同步并在历史里标记原因。               |
| 文件缓存配额（MB）   | 每台对端设备占用的本地缓存上限，超出按 LRU 清理。      |
| 文件保留时长（小时） | 缓存中的 blob 块在多久后允许被回收。                   |
| 自动清理             | 是否按上述阈值定期清缓存。                             |

按内容类型 / 按对端设备的细粒度开关 **不在这里**，而是在每台对端设备的[设备设置 modal](../core-features/devices#内容类型白名单)里。

## 安全

| 设置             | 作用                                                                               |
| ---------------- | ---------------------------------------------------------------------------------- |
| 自动解锁加密会话 | 锁屏 / 唤醒后是否自动用系统密钥环里的 KEK 解封 MasterKey。关掉则每次都要手动解锁。 |

更详细的密钥模型见 [配对与同步 — 信任](./pairing#信任空间--邀请码模型)。

## 网络

<Feature since="0.7.0">
  **仅局域网模式**、**允许覆盖网络地址** 与 **自定义中继节点** 都是 0.7
  新增的网络侧开关，修改后都需要重启 daemon 才会生效 —— 配套的「需要重启」 inline
  横幅只显示在你刚改过的字段上方。
</Feature>

| 设置             | 作用                                                                                                                                                                                                           |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 仅局域网模式     | 强制不走加密中继；跨网络对端会变不可达。同一局域网内的首次配对仍可工作（mDNS）。                                                                                                                               |
| 允许覆盖网络地址 | 打开后 Tailscale CGNAT (`100.64.0.0/10`) 与 IPv6 ULA (`fd7a:115c:a1e0::/48`) 才会进入邀请与本端发布的候选；关闭只保留物理网卡地址。Clash TUN fake-ip (`198.18.0.0/15`) 与 IPv4 link-local 始终被过滤、无开关。 |
| 自定义中继节点   | 每行一个 iroh relay URL。留空使用默认中继；非空时只使用这些自定义中继节点。                                                                                                                                    |

| 拥塞控制算法 | 网络流量控制算法。**CUBIC**（默认）在局域网和带宽充足的网络上表现最佳；**BBR3** 在高延迟或丢包较多的远程链路上可能更好，代价是局域网吞吐量降低。 |

任一项改完会显示一行横幅，提示需要重启后台服务。点`立即重启`即可——仅重启守护进程，应用界面保持不动。
系统托盘图标也会反映「仅局域网」当前是否开启。

要从零搭一台自己的中继节点（含 TLS、systemd / Docker、接入控制），见
[自建 iroh relay](./self-host-relay)。

## 存储

这是设置面板里最大、最容易踩的一组。

| 区块         | 内容                                                                                                                  |
| ------------ | --------------------------------------------------------------------------------------------------------------------- |
| 存储用量     | 数据库 / 加密 Vault / 缓存 / 日志四条进度条；`刷新`按钮强制重读。                                                     |
| 搜索索引     | 状态徽标（就绪 / 重建中 / 不可用）、上次重建时间、`重建`按钮。详见 [全文搜索](../core-features/search#索引生命周期)。 |
| 自动清理历史 | 总开关。关闭后只能手动清理。                                                                                          |
| 保留天数     | 7 / 30 / 90 / 180 / 365 几档。                                                                                        |
| 最大条目数   | 100 / 500 / 1000 / 2000 / 5000 几档。两个上限 **任一** 触发就清理。                                                   |
| 保留置顶     | 清理时跳过被置顶的条目。                                                                                              |
| 危险区       | `清空缓存`（删除文件块缓存，不影响历史）；`清空所有历史`（不可逆）；`打开数据目录`（直达本机存储位置）。              |

数据目录路径见 [安装 — 数据存储位置](../getting-started/install#数据存储位置)。

## 移动端同步

移动端同步的配置 **不在** 设置面板里。入口在 **设备** 页「手机同步」
tab 行右侧的 `配置` 按钮，点开会弹出居中的 **移动端同步设置** modal。

modal 里的字段速查：

| 设置                          | 作用                                                                                                                                          |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 启用移动端同步                | 功能总开关。关掉后没有监听器、没有 `Add device` 入口，LAN 端口不接受任何 HTTP 请求。                                                          |
| 启用 LAN 监听器               | 独立的 HTTP 监听器开关。需要总开关也开着才会生效。                                                                                            |
| 监听 / 对外宣告 IP            | 桌面让手机端连接时使用的 LAN IPv4。实际 socket 永远绑在 `0.0.0.0`，这一项只是改 URL。                                                         |
| 端口                          | 默认 **42720**（SPEC §3.2）。冲突时可改成自定义端口。                                                                                         |
| 当前监听地址（Listening URL） | 只读；从已保存设置直接拼出来，就是手机端要填的目标 URL。                                                                                      |
| 监听器 bind 错误              | 当守护进程上抛 `lan_listener_error`（端口被占用、权限不足等）时，modal 顶部会渲染一个红色 Alert 显示具体原因，便于排查为什么 URL 实际不可达。 |

<Callout type="info">
  「监听 / 对外宣告 IP」留作 `Auto`，dialog 的 **当前监听地址** 行会内联列出全部 RFC1918 候选 IP
  供复制；挑一个具体网卡则行内固定为单一 URL。实际 socket 永远 bind 在 `0.0.0.0`，这一项只决定
  对外宣告的地址。
</Callout>

完整的端到端配置流程、安全注意事项、故障排查请看
[移动端同步指南](../core-features/mobile-sync)。

## 关于

显示版本号、构建哈希、license 与项目链接。从这里也能跳到 [GitHub Releases](https://github.com/UniClipboard/UniClipboard/releases) 看更新日志。

### 更新设置

| 设置         | 作用                                                                                                                                                                                                                                                                                                                                                                  |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 自动检查更新 | 启动后由后台 scheduler 周期检查新版本（每 6 小时 ± 抖动），即使主窗口关闭、应用在后台闲置时也会照常检查；发现新版本会弹出紧凑的更新对话框，提供三个选项：**跳过此版本**、**稍后提醒**、**安装更新**。选择「跳过此版本」后 scheduler 不再为该版本弹窗；选择「稍后提醒」仅关闭对话框，下次检查仍会提醒。默认开启。关闭后只能从这里手动点 `检查更新`，也不会再自动弹窗。 |
| 后台下载更新 | 检查到新版本后静默下载安装包，下完点图标可一键安装。默认开启；依赖「自动检查更新」开启才能生效。更新对话框中也包含一个开关可以直接切换该设置。`.deb` / `.rpm` 安装会自动跳过该步骤 —— 这两种安装通过系统包管理器升级，提前下载安装包没有意义。                                                                                                                        |
| 更新频道     | `自动` / `稳定版` / `Alpha`。Alpha 是日常滚动版，切换前会有提醒。                                                                                                                                                                                                                                                                                                     |

侧边栏右下角的更新图标会反映当前状态：

- **琥珀色 + 闪烁圆点** —— 发现新版本，待操作。
- **琥珀色 + 进度环** —— 安装包正在后台下载，对话框里可取消。
- **绿色 + 钩号** —— 安装包已就绪，点击立即安装并重启。

## 当前 **没有** 的开关

下面这些功能在路线图里、或者只能曲线绕开，建议看到再回来对账：

- **关闭即驻留托盘 / 关闭即退出** —— 没有独立开关，关闭主窗口默认驻留后台。要彻底退出请用托盘菜单的`退出`。
- **通知偏好** —— 收到新剪贴时是否提醒，目前没有 UI 控制。
- **设备重命名（远端）** —— 只能在那台设备本机改，本机改完会同步到对端的成员列表。
- **整个空间的全局内容类型黑名单** —— 当前只能按对端设备粒度配置（[设备页](../core-features/devices)）。
- **永远拒走中继** —— "仅局域网模式"是最接近的，但它还会同时切掉跨网络对端。

如果上面有你强需求的，欢迎在 [github.com/UniClipboard/UniClipboard/issues](https://github.com/UniClipboard/UniClipboard/issues) 开个 issue 描述使用场景。
