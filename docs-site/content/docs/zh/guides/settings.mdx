设置面板按主题分为七组（外加一个「关于」）。下面按面板里的左右顺序逐组列出，每项给出 **作用** 和 **何时该改它**。

移动端同步的配置 **不在** 设置面板里，而是放在 **设备** 页的独立抽屉中
—— 单独整理在 [下文](#移动端同步) 与 [移动端同步](./mobile-sync) 一节。

## 通用

| 设置               | 作用                                                                        |
| ------------------ | --------------------------------------------------------------------------- |
| 开机自启           | 系统登录时自动启动 UniClipboard。                                           |
| 静默启动           | 启动后不弹窗，直接进入后台 / 托盘。                                         |
| 设备名             | 显示在其它设备的「成员列表」里 —— 起一个人能认的名字。                      |
| 语言               | 当前支持英文 / 简体中文。                                                   |
| 隐私：诊断数据     | 是否发送崩溃、错误、匿名日志与前端诊断数据；默认开启，可随时关闭。          |
| 隐私：使用情况分析 | 是否发送匿名产品事件（激活漏斗、配对 / 同步可靠性）；默认开启，可单独关闭。 |

设备名只是显示用的别名，不参与加密身份。改名不会影响 node ID 或配对状态。

诊断数据与使用情况分析是两个独立开关。它们不包含剪贴板内容、文件内容、口令或密钥；完整字段范围见 [隐私与数据收集](./privacy)。

## 外观

| 设置     | 作用                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 主题模式 | 浅色 / 深色 / 跟随系统。切换即时生效。                                       |
| 主题色   | 调色盘选择强调色（slate, rose, orange, amber, ..., blue, violet 等）。       |
| 界面缩放 | 80% / 90% / 100% / 110% / 120% 几档可选，也可拖滑块自由调；`重置`回到 100%。 |

缩放只影响渲染密度（DPI 概念），不影响数据；多设备之间不会同步。

## 快捷键

按"作用域"分组列出可绑定的命令，每一项都可以单独覆盖、重置或清空：

- **全局**：跨进程生效，包括 [快捷面板](./quick-panel) 的唤起键。
- **剪贴**：主仪表盘里浏览历史时的导航键。
- **设置**：设置面板内部导航。
- **设备**：设备页内的导航。
- **模态**：通用对话框 / 抽屉操作。

录制框直接按下你想用的组合即可，冲突会即时高亮提示。`重置`回默认，`清空`让该项无绑定。

## 同步

控制本机参与同步的方式与文件相关阈值：

| 设置                 | 作用                                                   |
| -------------------- | ------------------------------------------------------ |
| 启用自动同步         | 关闭后这台设备只接收，不主动推送（仍可手动 `send`）。  |
| 启用文件同步         | 总开关；关掉时文件类负载不再传出 / 传入。              |
| 小文件阈值（MB）     | 低于此阈值的文件按内联编码走快通道；高于的进 blob 流。 |
| 单文件最大值（MB）   | 超过此值会被拒绝同步并在历史里标记原因。               |
| 文件缓存配额（MB）   | 每台对端设备占用的本地缓存上限，超出按 LRU 清理。      |
| 文件保留时长（小时） | 缓存中的 blob 块在多久后允许被回收。                   |
| 自动清理             | 是否按上述阈值定期清缓存。                             |

按内容类型 / 按对端设备的细粒度开关 **不在这里**，而是在每台对端设备的[设备设置抽屉](./devices#内容类型白名单)里。

## 安全

| 设置             | 作用                                                                               |
| ---------------- | ---------------------------------------------------------------------------------- |
| 自动解锁加密会话 | 锁屏 / 唤醒后是否自动用系统密钥环里的 KEK 解封 MasterKey。关掉则每次都要手动解锁。 |

更详细的密钥模型见 [配对与同步 — 信任](./pairing#信任空间--邀请码模型)。

## 网络

<Feature since="0.7.0">
  **仅局域网模式** 与 **允许覆盖网络地址** 都是 0.7 新增的网络侧开关， 两者都需要重启 daemon
  才会生效 —— 配套的「需要重启」inline 横幅 也是 0.7 新增，只显示在你刚改过的字段上方。
</Feature>

| 设置             | 作用                                                                 |
| ---------------- | -------------------------------------------------------------------- |
| 仅局域网模式     | 强制不走加密中继；跨网络对端会变不可达。                             |
| 允许覆盖网络地址 | 允许从 rendezvous 拉取的发布地址参与候选；关掉只用 mDNS + 直连缓存。 |

任一项改完会显示一行黄色横幅，提示需要重启守护进程。点上面的`应用并重启`即可。
系统托盘图标也会反映「仅局域网」当前是否开启。

## 存储

这是设置面板里最大、最容易踩的一组。

| 区块         | 内容                                                                                                     |
| ------------ | -------------------------------------------------------------------------------------------------------- |
| 存储用量     | 数据库 / 加密 Vault / 缓存 / 日志四条进度条；`刷新`按钮强制重读。                                        |
| 搜索索引     | 状态徽标（就绪 / 重建中 / 不可用）、上次重建时间、`重建`按钮。详见 [全文搜索](./search#索引生命周期)。   |
| 自动清理历史 | 总开关。关闭后只能手动清理。                                                                             |
| 保留天数     | 7 / 30 / 90 / 180 / 365 几档。                                                                           |
| 最大条目数   | 100 / 500 / 1000 / 2000 / 5000 几档。两个上限 **任一** 触发就清理。                                      |
| 保留置顶     | 清理时跳过被置顶的条目。                                                                                 |
| 危险区       | `清空缓存`（删除文件块缓存，不影响历史）；`清空所有历史`（不可逆）；`打开数据目录`（直达本机存储位置）。 |

数据目录路径见 [安装 — 数据存储位置](../getting-started/install#数据存储位置)。

## 移动端同步

移动端同步的配置 **不在** 设置面板里。入口在 **设备** 页的「移动端同步」
面板：点 `Configure` 会从右侧滑出 **移动端同步设置** 抽屉。

抽屉里的字段速查：

| 设置                          | 作用                                                                                                      |
| ----------------------------- | --------------------------------------------------------------------------------------------------------- |
| 启用移动端同步                | 功能总开关。关掉后没有监听器、没有 `Add device` 入口，LAN 端口不接受任何 HTTP 请求。                      |
| 启用 LAN 监听器               | 独立的 HTTP 监听器开关。需要总开关也开着才会生效。                                                        |
| 监听 / 对外宣告 IP            | 桌面让手机端连接时使用的 LAN IPv4。实际 socket 永远绑在 `0.0.0.0`，这一项只是改 URL。                     |
| 端口                          | 默认 **42720**（SPEC §3.2）。冲突时可改成自定义端口。                                                     |
| 当前监听地址（Listening URL） | 只读；从已保存设置直接拼出来，就是手机端要填的目标 URL。                                                  |
| 监听器 bind 错误              | 只读；显示守护进程上抛的 `lan_listener_error`（端口被占用、权限不足等），便于排查为什么 URL 实际不可达。  |
| 「需要重启」提示横幅          | 当 daemon 在跑、但配置已变时出现。从托盘菜单退出再重新打开，或 `uniclip stop && uniclip start` 即可生效。 |

<Callout type="warn">
  「监听 / 对外宣告 IP」请挑一个 **具体的 LAN IPv4**。`Auto` / `0.0.0.0` 作为占位是 OK 的，但
  **不要** 把它印到手机屏幕上 —— 手机拨不通这个地址。 用下拉框挑一个手机会接入的 RFC1918 网卡 IP。
</Callout>

完整的端到端配置流程、安全注意事项、故障排查请看
[移动端同步指南](./mobile-sync)。

## 当前 **没有** 的开关

下面这些功能在路线图里、或者只能曲线绕开，建议看到再回来对账：

- **关闭即驻留托盘 / 关闭即退出** —— 没有独立开关，关闭主窗口默认驻留后台。要彻底退出请用托盘菜单的`退出`。
- **通知偏好** —— 收到新剪贴时是否提醒，目前没有 UI 控制。
- **设备重命名（远端）** —— 只能在那台设备本机改，本机改完会同步到对端的成员列表。
- **整个空间的全局内容类型黑名单** —— 当前只能按对端设备粒度配置（[设备页](./devices)）。
- **永远拒走中继** —— "仅局域网模式"是最接近的，但它还会同时切掉跨网络对端。

如果上面有你强需求的，欢迎在 [github.com/UniClipboard/UniClipboard/issues](https://github.com/UniClipboard/UniClipboard/issues) 开个 issue 描述使用场景。
