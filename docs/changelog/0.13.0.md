## 0.13.0 - 2026-06-01

### Breaking Changes

- Simplified the `uniclip mobile-sync` CLI by replacing the old `devices`, `settings`, and `lan` command groups with task-focused commands, so scripts using the old names need updating (#914)

### Features

- Mobile device pairing now opens on a scan-first QR flow, includes app download options, lets you switch the advertised address from the QR screen, and manages each paired device from one dialog (#883)
- First-time device pairing can now work on the same local network even when the internet pairing service is unreachable or LAN-only mode is enabled (#896)
- The tray menu now includes Check for Updates and Restart actions (#893)
- Mobile sync can now advertise a full public URL, making it possible to use a self-hosted server or secure proxy for cross-network phone access (#906)
- Advanced users can now run UniClipboard as a headless server node and deploy it on a VPS with the included Docker and Caddy setup (#910)
- Invite codes entered through the CLI are now case-insensitive and do not require hyphens (#917)
- Windows now offers portable x64 and arm64 downloads, with portable data stored beside the app and guided manual updates (#921)

### Fixes

- Fixed Linux keyring failures that could crash or block secure storage on some systems (#882)
- Windows no longer crashes when reading rich text copied from some Chinese-language Office or chat apps, and screenshot copying from modern apps is faster (#894)
- Update popups now behave as independent native windows instead of being minimized with the main window (#916)
- Launch at startup now actually registers with the operating system and repairs stale startup entries after moving or updating the app (#925)
- Update popups now come to the front on macOS even when the app was started silently or hidden in the background (#926)
- Fixed a Windows startup crash that could happen when the app was launched at login (#929)
- Automatic update checks now keep running while the app is idle in the background or resumes from sleep, so new-version prompts no longer wait until the main window is opened (#931)
