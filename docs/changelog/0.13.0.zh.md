## 0.13.0 - 2026-06-01

### Breaking Changes

- 简化了 `uniclip mobile-sync` 命令行用法，旧的 `devices`、`settings` 和 `lan` 命令组已替换为按任务组织的新命令，因此使用旧命令的脚本需要更新 (#914)

### Features

- 移动设备配对现在默认进入扫码流程，内置客户端下载入口，可在二维码界面切换对外地址，并可在同一个对话框里管理每台已配对设备 (#883)
- 首次设备配对现在即使在互联网配对服务不可用或启用 LAN-only 模式时，也可以在同一局域网内完成 (#896)
- 托盘菜单新增检查更新和重启入口 (#893)
- 移动端同步现在可以对外提供完整公开 URL，便于通过自托管服务器或安全代理实现跨网络手机访问 (#906)
- 高级用户现在可以将 UniClipboard 作为无界面的服务器节点运行，并使用内置 Docker 与 Caddy 配置部署到 VPS (#910)
- 通过命令行输入的邀请码现在不区分大小写，也不再要求必须带连字符 (#917)
- Windows 现在提供 x64 和 arm64 便携版下载，便携版数据会保存在应用旁边，并提供手动更新引导 (#921)

### Fixes

- 修复部分 Linux 系统上钥匙串故障可能导致崩溃或安全存储不可用的问题 (#882)
- Windows 读取部分中文 Office 或聊天软件复制的富文本时不再崩溃，并提升了从现代应用复制截图的速度 (#894)
- 更新弹窗现在是独立的原生窗口，不会再跟随主窗口一起最小化 (#916)
- 开机启动开关现在会真正注册到操作系统，并会在应用移动或更新后修复失效的启动项 (#925)
- macOS 上即使应用静默启动或隐藏在后台，更新弹窗也能正常显示到前台 (#926)
- 修复 Windows 登录后自动启动时可能崩溃的问题 (#929)
- 自动检查更新现在会在应用后台空闲或从睡眠恢复后继续运行，新版本提示不再等到打开主窗口后才出现 (#931)
