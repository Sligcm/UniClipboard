## 0.14.0 - 2026-06-10

### Breaking Changes

- The sync engine now ships as a separate `uniclipd` program: the `uniclip daemon` subcommand is removed and `uniclip start` launches `uniclipd` automatically, so scripts or server setups that invoked `uniclip daemon` directly need updating (#960)

### Features

- Clipboard sync now runs as a standalone background service, with a new Lightweight Mode in the tray menu: the app window can fully exit while syncing keeps running in the background, and choosing Quit in the tray stops both the app and the sync service (#960)
- The `uniclip` command line now talks to the same background service as the desktop app, so both always see the same clipboard state, and CLI commands work even when the desktop app is not running (#975)
- Add a quick panel setting for opening at the screen center or near the cursor, and make filtering from the search bar easier to use by mouse or keyboard (#936)
- Add a compact update dialog with options to skip a version, be reminded later, install the update, and toggle future automatic downloads (#992)
- Apply network setting changes by restarting the background service while keeping the app window open, with clearer progress and success feedback (#994)
- Sending content from mobile that already exists on the desktop now makes it immediately available to paste again (#1003)

### Fixes

- Make UniClipboard appear correctly in Linux application menus instead of being hidden or grouped under Other (#934)
- Fix the macOS Dock icon so it no longer appears blank, oversized, or surrounded by an unwanted white ring (#935)
- Restore the Linux AppImage download for users who install UniClipboard that way (#944)
- Prevent repeated image captures from creating duplicate history entries, repeated syncs, or unbounded cache growth (#959)
- Load large image previews in history on demand so browsing image-heavy history no longer drives memory usage up (#960)
- Restore Snap package availability for Linux users (#968)
- Prevent the background service from appearing as a bouncing app icon in the macOS Dock (#969)
- Fix mobile sync device removal and password rotation so they no longer fail with a not-found error (#972)
- Show an actionable error screen instead of a blank loading screen when the app cannot connect to the background service (#973)
- Stop the old background service before installing an update so the installer can replace it cleanly, especially on Windows (#977)
- Recover startup when an incompatible background service is running but its process record is missing (#980)
- Use a monochrome menu bar icon on macOS so the tray icon matches light and dark menu bar appearances (#974)
- Show the saved mobile device name for clipboard items received from a phone instead of a technical device ID (#984)
- Restore the normal full-color tray icon on Windows and Linux (#983)
- Fix VPS Docker deployments so the headless server can start instead of crash-looping (#986)
- Prevent the app from getting stuck on the loading screen after the computer wakes or reconnects, and let users retry without restarting (#999)
- Restarting the desktop background service after settings changes no longer leaves clipboard history locked or blocks follow-up actions (#1002)
- Prevent iPhone sync from repeatedly pulling the same image or file as new content (#1007)
- Show a clear error when the app cannot finish startup instead of leaving setup stuck loading (#1009)
