## 0.14.0 - 2026-06-10

### Breaking Changes

- 同步引擎现在作为独立的 `uniclipd` 程序发布：`uniclip daemon` 子命令已移除，`uniclip start` 会自动拉起 `uniclipd`；如有脚本或服务器部署直接调用过 `uniclip daemon`，需要相应更新 (#960)

### Features

- 剪贴板同步现在由独立的后台服务承担，并在托盘菜单新增「轻量模式」：应用窗口可以完全退出，同步继续在后台运行；在托盘选择「退出」则会连同后台服务一起停止 (#960)
- `uniclip` 命令行改为与桌面应用共用同一个后台服务，两边看到的剪贴板状态始终一致，不打开桌面应用也能直接使用命令行 (#975)
- 新增快捷面板展示位置设置，可选择在屏幕中央或鼠标附近打开，并让搜索栏筛选更适合鼠标和键盘操作 (#936)
- 新增更紧凑的更新对话框，可跳过某个版本、稍后提醒、安装更新，并可直接切换今后是否自动下载更新 (#992)
- 网络设置变更现在会重启后台服务来生效，同时保持应用窗口打开，并提供更清晰的进度和成功反馈 (#994)
- 从手机发送桌面端已存在的内容时，现在会让这条内容重新变成可直接粘贴的内容 (#1003)

### Fixes

- 修复 UniClipboard 在 Linux 应用菜单中可能被隐藏或归到“其他”分类的问题 (#934)
- 修复 macOS Dock 图标空白、过大或带有异常白色圆环的问题 (#935)
- 恢复面向通过 AppImage 安装 UniClipboard 的 Linux 用户的下载包 (#944)
- 修复重复图片捕获可能造成剪贴板历史重复、反复同步或缓存持续增长的问题 (#959)
- 历史记录中的大图预览改为按需加载，浏览大量图片时内存占用不再持续升高 (#960)
- 恢复 Linux 用户可用的 Snap 软件包发布 (#968)
- 修复后台服务在 macOS Dock 中显示为持续跳动图标的问题 (#969)
- 修复移动同步中移除设备和轮换密码会返回未找到错误的问题 (#972)
- 当应用无法连接后台服务时，显示可操作的错误页面，而不是停在空白加载界面 (#973)
- 安装更新前会先停止旧的后台服务，确保安装器能顺利替换文件，尤其是在 Windows 上 (#977)
- 修复已有不兼容后台服务在缺少进程记录时导致应用启动无法恢复的问题 (#980)
- macOS 菜单栏图标改为单色样式，使托盘图标能匹配浅色和深色菜单栏外观 (#974)
- 从手机接收的剪贴板内容现在会显示已保存的移动设备名称，而不是技术性的设备 ID (#984)
- 恢复 Windows 和 Linux 上正常的全彩托盘图标 (#983)
- 修复 VPS Docker 部署中无界面服务无法启动、容器反复崩溃的问题 (#986)
- 修复电脑唤醒或重新联网后应用可能卡在加载页的问题，并允许用户无需重启即可重试 (#999)
- 修改设置后重启桌面后台服务，不会再导致剪贴板历史保持锁定或后续操作被阻止 (#1002)
- 修复 iPhone 同步时会把同一张图片或同一个文件反复当作新内容拉取的问题 (#1007)
- 当应用无法完成启动时显示清晰错误，不再让设置流程一直停在加载中 (#1009)
