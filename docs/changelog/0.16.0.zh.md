## 0.16.0 - 2026-06-22

### Features

- 新增 Debug 模式和一键导出日志，可从设置或命令行收集排障信息 (#1087)
- 新增官方 Homebrew cask 支持，方便在 macOS 上安装和更新 UniClipboard (#1086)
- 新增 Chocolatey、winget、Scoop、Flathub 和 Nix 的包管理器安装支持 (#1088)
- 新增配置备份与导入，可从设置或首次启动流程把已有 UniClipboard 配置迁移到另一台设备 (#1118)
- 新增“恢复时同步”选项，可在从历史记录恢复内容时把它同步为其他设备的当前剪贴板 (#1117)
- 新增网络传输行为设置，局域网推荐使用 CUBIC，也可为远距离或丢包较多的链路选择 BBR3 (#1121)

### Fixes

- 修复 Snap 包安装后缺少预期二进制文件的问题 (#1075)
- 减少 Linux/X11 复制图片后可能反复捕获、刷屏剪贴板历史的问题 (#1083)
- 修复命令行 `uniclip get` 在终端输出文本时不会换到新行的问题 (#1091)
- 修复 Linux 剪贴板图片读取会优先选择不可用图片目标的问题 (#1089)
- 修复 Linux/X11 剪贴板读取可能在剪贴板所有者尚未完成提供内容前就放弃的问题 (#1096)
- 修复 Linux 共享剪贴板图片因仅 alpha 通道变化而被反复捕获的问题 (#1105)
- 修复开启 Debug 模式重启时对话框过早关闭、用户看不到仍在重启的问题 (#1102)
- 修复 macOS 上再次复制相同文本时，剪贴板历史不会把该项目重新置顶的问题 (#1112)
- 修复跨设备文件同步的识别方式，避免同一个复制文件被反复拉取 (#1126)
- 当对端离线导致剪贴板条目无法送达时，现在显示为不可达而不是失败 (#1131)
- 修复 macOS 和 Windows 上并发文件传输可能把缓存文件变成空文件的问题 (#1132)
- 修复同一个复制文件通过重叠同步路径到达时产生重复条目和重复下载的问题 (#1133)
