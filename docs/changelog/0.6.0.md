## 0.6.0 - 2026-05-02

### Breaking Changes

- This release rebuilds how devices connect to each other. After upgrading from 0.5.0, existing pairings are invalidated and every device must be paired again before syncing can resume.

### Features

- More reliable cross-network sync: connection success rates are noticeably higher across different Wi-Fi networks, home, and office setups, and recovery is faster on flaky or briefly degraded networks.
- Instant resume after sleep or network changes: after sleep/wake, switching Wi-Fi, or a brief disconnect, syncing continues without a fresh handshake — no more "I copied something but it took a while to arrive."
- Clearer first-run setup: on first launch you can create a new personal space, or join an existing one with an invitation code and passphrase.
- Improved Devices page: generate a short-lived invitation code in one click, copy or regenerate it, and see in real time whether the other device joined successfully.
- Join another space without losing history: you can join a different space from the Devices page, and your local clipboard history is preserved and migrated.
- Better local-network discovery so pairing and transfers are more reliable when devices are on the same Wi-Fi.
- Upgrade notice: when upgrading to this version, a dedicated prompt tells you that re-pairing is required, so the upgrade no longer fails silently.
- Expanded CLI: new `uniclip init` to create a space, `uniclip invite` to generate an invitation code, `uniclip join` to join a space, `uniclip members` to list paired devices and presence, `uniclip send` / `uniclip watch` to send or observe clipboard content directly from the terminal, `uniclip switch-space` to switch an already-paired device to another space (with local history migrated automatically), and `uniclip upgrade status / ack` to inspect and acknowledge upgrade prompts. The full setup, pairing, and sync flow can now be completed from the terminal without opening the GUI.

### Fixes

- Fixed the paired-devices list and count so the current device is no longer counted as a paired device.
- Fixed invitation-code entry so pasted codes are accepted even when separators are missing.
- Fixed duplicate clipboard-history entries that could appear after receiving content from another device.
- Fixed cases where file lists, PNG images, and rich clipboard content could lose formats when written to the system clipboard on Windows or macOS.
- Fixed occasional hangs when quitting or restarting the app.
- Prevented the add-device dialog from replacing the success state with a new invitation after pairing completes.
- Removing a device now fully clears it so it no longer receives sync attempts and can be paired again cleanly.
- New installs no longer show an incorrect re-pair prompt after the first setup is completed.
- Avoid unusable virtual network addresses so device connections and transfers are less likely to fail on VPN or proxy setups.
- Prevent clearing cache on Windows from removing saved setup, settings, and history data.
- Keep file and image clipboard history more stable when cached files are deleted or expire.
- Resume clipboard sync automatically after the background service restarts.
- Show paired devices with their real online state and update offline status sooner while the Devices page is open.
- Fix the quick panel opening to a blank window.
- Improved overall sync reliability after cold starts, brief device offline periods, and network changes.
