## 0.7.0-alpha.7 - 2026-05-10

### Features

- Mark history items whose saved content is gone as unavailable in the main history and quick panel before users try to reuse them.

### Fixes

- Show a clear unavailable-content message for history items that can no longer be restored instead of a generic copy failure.
- Finish saving large clipboard captures before adding them to history, reducing cases where an item appears after restart but cannot be reused.
- Detect older broken history items on startup and mark them unavailable so repeated reuse attempts no longer produce generic failures.
