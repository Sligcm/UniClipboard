## 0.7.0 - 2026-05-10

### Features

- New LAN-only Mode in Network settings keeps clipboard sync entirely on the local network. The settings page shows a restart notice and lists exactly what can still reach the internet while it is on, and the system tray reflects whether LAN-only Mode is active.
- Device cards now show how each paired device is reachable: directly, relayed, disconnected, or out of reach on the local network when LAN-only Mode is on.
- Per-device sync gates: choose whether each paired device sends, receives, and which content types (text, images, files) are allowed.
- Mobile sync via iPhone Shortcut over the local network, with QR-code pairing, separate iOS and Android setup instructions, and a credentials view that supports username display and password rotation without removing the device.
- History items whose underlying content is gone are now greyed out as unavailable in both the main history and the Quick Panel, with a clear unavailable-content message instead of a generic copy failure.
- VPN users can opt in to using virtual network addresses when both devices reach each other through the same virtual network.
- Experimental network settings now carry a clear Experimental badge and tooltip.
- Linux now ships RPM packages with a Fedora COPR install channel (`dnf install uniclipboard`), plus a strict Snap package that covers older distributions.
- Paired desktops return to Online sooner after reconnecting, while unreachable peers are checked less aggressively.

### Fixes

- Existing trusted devices are preserved when upgrading from 0.6.x to 0.7.x — pairing carries over without a manual re-pair.
- Settings files written by earlier versions no longer prevent the app from starting after an upgrade.
- LAN-only Mode now truly keeps already-paired devices on local-network discovery and direct connections instead of still reaching them through public network services.
- Search now matches partial terms inside copied addresses and code-style text such as `localhost`, IP addresses, and mixed-case identifiers.
- Quick Panel paste on macOS now reliably returns focus to the previous app before pasting the selected item.
- Rich text and HTML clipboard history entries restore without falling back to plain text or truncated content.
- Reduced repeated macOS Keychain prompts during startup, setup, and unlock.
- The macOS Dock icon once again restores the main window after it has been closed to the tray.
- The Network settings page keeps restart notices and error messages in sync with the actual saved value.
- The telemetry switch now controls all optional diagnostics immediately, without requiring an app restart.
- Entering the wrong passphrase while joining a space no longer breaks the existing setup or forces a reset.
- Per-device sync switches save instead of reverting, and the per-device send / receive / content-type filters now actually take effect.
- Clipboard copies made right after unlocking are no longer missed while peers reconnect.
- Large clipboard captures finish saving before being added to history, so they can be reused after a restart instead of producing a generic copy error.
- Older broken history items are detected on startup and marked unavailable up front.
- Images sent from mobile clients now arrive as images even when the client does not provide a specific image type.
- Incoming clipboard items no longer leave a stuck receiving placeholder after the content has arrived.
- Window spacing, dialog overlays, and minimum sizes fit better with the redesigned title bar.
