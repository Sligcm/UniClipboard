## 0.7.0 - 2026-05-10

### Features

- 网络设置新增「仅局域网」模式，开启后剪贴板同步完全限制在局域网内。设置页带重启提示，并明确列出此模式下仍会访问公网的项；系统托盘也会显示当前是否处于「仅局域网」状态。
- 设备卡片现在会显示每台已配对设备的连接通道：直连、中继、离线，或在仅局域网模式下「超出局域网范围」。
- 每设备同步开关：可以分别选择某台设备是否发送、接收，以及允许哪些内容类型（文本 / 图片 / 文件）。
- 通过 iPhone Shortcut 进行局域网剪贴板同步，支持二维码配对、iOS 与 Android 分别的接入指引，以及无需重新配对即可查看用户名和重置密码的凭据视图。
- 历史中底层内容已丢失的条目会在主列表和快捷面板中置灰为「不可用」，并给出清晰提示，不再表现为通用的复制失败。
- 同时使用 VPN 时，可选择允许使用虚拟网络地址，让两端走同一虚拟网络互通。
- 实验性的网络设置统一带 Experimental 徽章和提示，避免误触。
- Linux 新增 RPM 安装包和 Fedora COPR 渠道（`dnf install uniclipboard`），并补齐 strict Snap 包以覆盖更老的发行版。
- 已配对的桌面端在重连后能更快回到 Online，无法连通的对端也不再被频繁探测。

### Fixes

- 从 0.6.x 升级到 0.7.x 时，原有的已信任设备会被保留，无需重新配对。
- 旧版本写入的设置文件不会再导致升级后启动失败。
- 「仅局域网」模式下，已配对设备会真正限制在局域网发现和直连上，不再绕回公网服务。
- 搜索现在能按前缀命中复制内容中的地址和代码风格文本，比如 `localhost`、IP 地址、混合大小写的标识符。
- macOS 上从快捷面板粘贴时会先稳定切回原应用再粘贴所选条目，避免内容丢失。
- 富文本和 HTML 类型的剪贴板历史能恢复成原本格式，不再降级成纯文本或截断的内容。
- 减少 macOS 启动、初次设置和解锁时重复弹出的 Keychain 授权窗口。
- 主窗口被收到托盘后，点击 macOS Dock 图标能再次唤起主窗口。
- 网络设置页保存后，重启提示与错误信息会和当前实际保存的设置保持一致。
- 遥测开关现在会立即控制所有可选诊断上报，不再需要重启 App。
- 加入空间时输错口令不会再破坏已有 setup，也不会强制重置。
- 每设备同步开关会真正保存而不是回弹；按设备的发送 / 接收 / 内容类型过滤现在能实际生效。
- 解锁 App 后立刻复制的内容不会再因对端正在重连而丢失。
- 较大的剪贴板内容会先完成持久化再加入历史，避免重启后该条目无法再次使用、只能回提通用错误。
- 启动时会检出更早版本遗留的损坏历史条目并直接标记为不可用，避免反复尝试触发通用错误。
- 移动端发来的图片即便没有携带具体图片类型，现在也会以图片形式送达。
- 接收完成后不会再留下卡住的「接收中」占位条目。
- 配合标题栏重设计，整体调整了窗口边距、对话框遮罩和最小窗口尺寸。
